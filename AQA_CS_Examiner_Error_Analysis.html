How to Use This Document

This document summarises the most common errors and omissions identified by AQA examiners across three exam series (2022–2024) for both Paper 1 and Paper 2 of the AQA A-level Computer Science specification (7517). Each finding is cross-referenced with the relevant specification section.

Section 2 · Paper 1

Paper 1 – Topic Findings

🗃️
Data Structures
Spec 4.2
  • The circular queue deletion algorithm is asked regularly and the empty-queue check is frequently omitted. This has appeared in every exam series reviewed.
  • Binary tree properties must be specific to binary trees (≤2 children per node); generic tree properties earn no marks.
  • In-order traversal uses a stack — only one-third of students knew this.
  • Hash tables: explain the hash function, uniqueness of key, and collision handling — all three.
🧠
Algorithms, Complexity & Computability
Spec 4.4
  • The Halting problem was not known by more than half the cohort. It is non-computable — distinct from intractable (solvable but not in polynomial time).
  • Heuristics ≠ decomposition. Heuristics find a good-enough solution in reasonable time.
  • Constant time O(1) means the number of steps does not change with input size — not that execution time is constant.
  • Regular expressions: the alternation symbol | was unknown; (bb)* and bb* are not equivalent.
  • Set theory: the empty set ∅ is always a subset of every set; sets are unordered.
🔧
Object-Oriented Programming
Spec 4.3.3
  • Protected is the most misunderstood access specifier — it is not private; it is accessible in subclasses.
  • Method overriding answers must link the concept to inheritance.
  • Abstract methods must be overridden; virtual methods can be overridden — know the distinction.
  • Encapsulation is the bundling of data and methods, not the same as concatenation.
Section 2 · Paper 2

Paper 2 – Topic Findings

🔢
Data Representation
Spec 4.5 & 4.6
  • Colour depth = log₂(number of colours) — not the square root.
  • File size: divide by 8 (bits→bytes), divide by 1024 (not 1000) for binary prefixes.
  • Floating-point negatives: never write -N + (+fraction). Convert the positive value first, then negate.
  • Binary point shift direction: a negative exponent shifts the point LEFT.
  • Normalisation maximises precision and gives a unique representation — state both points.
  • Overflow = too large; underflow = too close to zero. Do not confuse with stack overflow.
⚙️
Hardware & Processor Architecture
Spec 4.7
  • I/O controller: say it translates signals between CPU and peripherals — not just 'interface'.
  • OS functions: naming 'memory management' is not enough — describe that the OS allocates memory regions to processes.
  • Fetch-execute cycle: describe all three stages — Fetch, Decode, AND Execute. Execute is most often omitted.
  • Interrupts pause (not stop) the FEC; the processor saves state.
  • Control bus ≠ control unit. Different things.
🌐
Networks, Internet & Security
Spec 4.8 & 4.9
  • Start bit: synchronises receiver's clock — not just 'marks the start'.
  • Stop bit: lets receiver prepare for the next start bit — not just 'marks the end'.
  • Transport layer: segmentation, port numbers, end-to-end connection. Routing is the network layer.
  • Gateway repackages data between incompatible network protocols — a router cannot do this.
  • NAT uses a translation table with both IP addresses and port numbers. Fewer than 10% could fully describe port forwarding.
  • Asymmetric encryption: both sender AND receiver each have their own key pair. Know which key is used when.
🗃️
Databases & SQL
Spec 4.9.3
  • CREATE TABLE: field_name data_type — not data_type field_name. Commas between field definitions.
  • UPDATE syntax: UPDATE table_name SET … — table name, not field name, follows UPDATE.
  • Date values require single quotation marks in INSERT/DELETE.
  • CRUD/REST: GET→SELECT, POST→INSERT, PUT→UPDATE, DELETE→DELETE.
  • Normalisation: describe the problem in practice — not just the label (e.g. 'data inconsistency').
  • Timestamp ordering: each record holds read/write timestamps used for concurrency control — very poorly understood.
🔣
Boolean Algebra & Logic
Spec 4.3
  • AND (∙) has higher precedence than OR (+). Do not simplify A + Ā to 1 if A is part of an AND expression.
  • De Morgan's law is only valid when all operators in a group are the same type (all AND or all OR).
  • D-type flip-flop: output is set to the DATA input at the clock edge — it does not toggle.
💻
Assembly Language
Spec 4.9.4
  • LSR shifts bits to the RIGHT — read the mnemonic carefully in traces.
  • Assembly language does need translation — by an assembler. It is not directly executable.
  • Immediate addressing: use # before a literal value. Omitting # uses direct addressing.
  • Do not store fractional values in a register unless the ISA explicitly supports it.
🧮
Functional Programming
Spec 4.9.5
  • Single-word answers like 'immutable' earn no marks — explain what immutability means.
  • Statelessness: functions produce the same output for the same inputs and do not modify external state.
  • Distributed computing link: if data cannot change and functions have no side-effects, multiple servers can process simultaneously without synchronisation errors.
  • Map always returns a list. Include brackets in written output.
Section 3

Exam Tips to Address Common Issues

📌 General Exam Technique – Across All Papers
Read the question carefully – answer what is asked, not a general description of the topic.
Address the context: if a scenario is given (e.g. file server, student house), tailor your answer to that context.
Do not repeat information already given in the question – it will not earn marks.
Show all working clearly for numerical questions so that method marks can be awarded even if the final answer is wrong.
Use precise technical vocabulary at A-level standard (e.g. 'RAM' not 'memory'; 'IP address' not 'address').
Attempt all questions – especially the longer programming tasks on Paper 1. Marks are available for simple, partial solutions.
Match the depth of your answer to the marks available. A 1-mark question needs one clear point; a 3-mark question needs three distinct points.
Explain why, not just what – naming a function or feature without describing what it does will rarely earn marks.
🔢 Data Representation (Paper 2 – Spec 4.5 & 4.6)
Colour depth: colour depth = log₂(number of colours). Do not take the square root.
File size conversions: remember to divide by 8 to convert bits to bytes, and by 1024 (not 1000) for kibibytes/mebibytes/gibibytes.
Memory addressing: the number of addressable locations is 2^(number of address bus lines) – use the bus width, not the word size.
Floating point – direction: when the exponent is negative, shift the binary point to the LEFT.
Floating point – negative numbers: convert the positive value to binary first, then negate using two's complement. Never write -23 then +0.25.
Normalisation: a normalised floating-point number maximises precision and ensures a unique representation. State both points in exam answers.
Floating point error types: know overflow (too large) and underflow (too close to zero) by name. Do not confuse with stack overflow.
Show every step of binary conversion working – partial credit is awarded for correct method even if the final answer is wrong.
⚙️ Hardware, Processors & Buses (Paper 2 – Spec 4.7)
Address bus: the number of wires determines addressable locations (2ⁿ), not the word size.
I/O controller: it translates signals between the CPU and peripherals – say 'translates', not just 'acts as an interface'.
Control bus vs control unit: the control bus carries control signals; the control unit manages instruction execution.
Fetch-execute cycle: describe all three stages – Fetch, Decode, AND Execute. The execute stage is most often omitted.
Performance improvements: explain the mechanism (e.g. 'multiple cores allow instructions to execute simultaneously').
Interrupts: an interrupt causes the processor to pause the current task and save state, not stop execution permanently.
Operating system functions: naming a function is not enough. Describe what the OS does, e.g. 'allocates regions of memory to different processes'.
Synchronous transmission: the clocks of transmitter and receiver are kept continuously synchronised – usually by a shared clock signal.
🌐 Networks, Internet & Security (Paper 2 – Spec 4.8 & 4.9)
Start bit: its purpose is to synchronise the receiver's clock with the sender's – not just to 'mark the start' of transmission.
Stop bit: allows the receiver to prepare for the next start bit – not just to 'mark the end'.
Transport layer: describes segmentation, port numbers, and end-to-end connection establishment – not routing (that is the network layer).
Gateway: a gateway can repackage data when connecting networks that use different protocols. A router cannot do this.
NAT: a NAT router uses both IP addresses AND port numbers (stored in a NAT translation table) to route return packets correctly.
Port forwarding: the router changes the destination IP address so that external packets can reach an internal server.
DNS: the purpose is to resolve fully qualified domain names to IP addresses. Describe the hierarchical structure and delegation.
Asymmetric encryption: BOTH the sender and receiver each have their OWN public and private key pair. Know which key is used at each step.
Digital signatures: signed with sender's PRIVATE key, verified with sender's PUBLIC key.
Vernam cipher: the KEY (not 'it') must be truly random, at least as long as the message, and used only once.
Parity bits: can detect an ODD number of changed bits – an even number of changed bits will not be detected.
🗃️ Databases & SQL (Paper 2 – Spec 4.9.3)
SQL CREATE TABLE: syntax is field_name data_type, not data_type field_name. Use commas between field definitions.
SQL UPDATE: syntax is UPDATE table_name SET … – the table name comes after UPDATE, not the field name.
SQL INSERT INTO: enclose date and string values in single quotation marks.
SQL DELETE: only include tables that are needed. A simple date-based delete does not require joining other tables.
CRUD and REST: GET→SELECT, POST→INSERT, PUT→UPDATE, DELETE→DELETE.
Normalisation problems: DESCRIBE the problem in practice, e.g. 'if a supplier address is stored in every product row, updating it requires changing every row, risking inconsistency'.
Timestamp ordering: each transaction gets a timestamp; each record holds a read-timestamp and write-timestamp used to detect conflicts.
🔣 Boolean Algebra & Logic (Paper 2 – Spec 4.3)
Operator precedence: AND (∙) has HIGHER precedence than OR (+). Do NOT simplify A + Ā to 1 if A is part of an AND expression.
De Morgan's law: can only be applied to a group of operators if they are ALL the same type (all AND or all OR). Do not mix.
De Morgan's forms: NOT(A AND B) = NOT A OR NOT B; NOT(A OR B) = NOT A AND NOT B.
Edge-triggered D-type flip-flop: the output is set to the value of the DATA input at the moment of the clock edge – it does not toggle.
Boolean simplification: show every step clearly. Marks are awarded for valid intermediate steps even if the final answer is wrong.
💻 Assembly Language (Paper 2 – Spec 4.9.4)
Shift direction: LSR (Logical Shift Right) shifts bits to the RIGHT – read the mnemonic carefully in traces.
Immediate addressing: use # before a literal value (e.g. ADD R0, R0, #48) – omitting # uses direct addressing.
Branch behaviour: code after a label executes sequentially – a branch to a label does NOT skip the code after that label automatically.
Assembly vs high-level: assembly language DOES need translation (by an assembler). Advantages are faster/smaller object code and direct hardware access.
Registers are integers: do not store fractional values in a register during a trace unless the ISA explicitly supports it.
🧮 Functional Programming (Paper 2 – Spec 4.9.5)
Immutability: means once a data structure is assigned a value, that value CANNOT be changed. State this clearly.
Statelessness: functions produce the same output for the same inputs and do not depend on or modify external state.
Distributed computing: if data cannot change and functions have no side-effects, multiple servers can process data simultaneously without synchronisation errors.
Map function: the result of map on a list is always another list. Include brackets when writing list output.
Recursive trace: follow the call stack carefully – work out what each recursive call returns before combining with the accumulated result.
🌲 Data Structures (Paper 1 – Spec 4.2)
Circular queue deletion: ALWAYS check if the queue is empty before removing an item. This check is frequently omitted.
Circular queue vs linear queue: in a circular queue, the front/rear pointers wrap around; no items are shuffled and there are no unused inaccessible locations.
Binary tree properties: a binary tree has at most TWO children per node – state this explicitly to distinguish it from a general tree.
Stack in traversal: in-order traversal of a binary tree uses a stack to track the current position. Know this and be able to describe it.
Hash tables: explain (1) the hash function produces the storage location, (2) the key must be unique, and (3) how collisions are handled.
Reverse Polish Notation: operands stay in their original order – do not change the order of A, B, C when converting. Only operators move.
🧠 Algorithms & Complexity (Paper 1 – Spec 4.4)
Halting problem: a non-computable problem for which no algorithm can ever be written, for any possible input. It is NOT the same as an intractable problem.
Intractable vs non-computable: intractable problems CAN be solved algorithmically but not in reasonable time. Non-computable problems CANNOT be solved at all.
Constant time complexity O(1): the NUMBER OF STEPS does not change with input size – not that the time taken is always the same.
Heuristics: an approach that finds a good-enough solution in reasonable time when an optimal algorithm is too slow. Not the same as decomposition.
Sets – subsets: the empty set ∅ is ALWAYS a subset of every set. Order does not matter in sets: {a, b} = {b, a}.
Regular expressions: | is the alternation (OR) symbol. * means zero or more. Brackets matter: (bb)* ≠ bb*.
🔧 Object-Oriented Programming (Paper 1 – Spec 4.3.3)
Protected access: a protected attribute/method is accessible within the class AND in subclasses. It is not the same as private.
Method overriding: a subclass provides its own implementation of a method defined in the superclass. Always link overriding to inheritance.
Abstract methods: MUST be overridden in any concrete subclass. A virtual method CAN be overridden but does not have to be.
Encapsulation: the bundling of data and methods into a class, and hiding the internal state from outside. Not the same as concatenation.
