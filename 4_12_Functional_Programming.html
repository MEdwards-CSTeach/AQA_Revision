➡️ Function Types & Notation

4.12.1.1

The Arrow Notation

f : A → B
f  =  the function name
A  =  the DOMAIN   (the set of valid inputs)
B  =  the CO-DOMAIN  (the set from which outputs are chosen)

Domain

The set of ALL valid inputs

Co-domain

The set from which outputs come (not every value need be produced)

Range

Only the values actually output (a subset of the co-domain)

Examples

square : integer → integer
square x = x * x

add : integer → integer → integer
(takes two integers, returns one integer)

f : {a,b,c…z} → {0,1,2…25}
(maps letters to numbers 0-25)
⚠️ Common exam mistakes
  • Confusing CO-DOMAIN with RANGE: Co-domain is the possible outputs; Range is the actual outputs produced.
  • If a function returns integers, the co-domain is "the set of integers" — write that, not "integer → integer".

🎯 First-Class Objects & Partial Application

4.12.1.2 – 4.12.1.4

First-Class Objects

A function is a "first-class object" if it can:

  • Appear in an expression
  • Be assigned to a variable
  • Be passed as an argument to another function
  • Be returned as the result of a function call

💡 This is what makes higher-order functions possible!

Partial Function Application

Supplying arguments ONE AT A TIME rather than all at once.

-- Full application:
add 3 4   →  7

-- Type of add:
add : integer → integer → integer

-- Partial application:
add 4  →  returns a NEW FUNCTION
          that adds 4 to any integer

addFour = add 4
addFour 10  →  14
⚠️ Exam tip Partial application RETURNS A FUNCTION, not a value. This is a common point of confusion. Make sure you can describe what "add 4" gives back.

🔗 Function Composition

4.12.1.5

Composition means applying one function to the result of another to create a new function.

The Notation: g ∘ f

Given:  f : A → B   and   g : B → C
g ∘ f  means: apply f FIRST, then g
(g ∘ f)(x)  =  g( f(x) )
Domain of g ∘ f is A
Co-domain of g ∘ f is C

Worked Example

f(x) = x + 2
g(y) = y³
g ∘ f = ?
Step 1: Apply f first  →  x + 2
Step 2: Apply g to result  →  (x + 2)³
So:   g ∘ f(x) = (x + 2)³
💡 Think of it as a pipeline: Input (Domain A) → f applied first → Intermediate (set B) → g applied second → Output (Co-domain C)

🔧 Higher-Order Functions: Map, Filter, Fold

4.12.2

A function is higher-order if it takes a function as an argument, OR returns a function as a result, OR both.

MAP

Applies a function to EVERY element in a list. Returns a new list of the same length.

square x = x * x
nums = [1, 3, 5]
map square nums
→  [1, 9, 25]

💡 Think: transform every item

FILTER

Keeps only elements that satisfy a condition. Returns a new list — possibly shorter.

b = [1, 5, 10, 15]
filter (<10) b
→  [1, 5]
Only items < 10 kept.

💡 Think: sieve / keep matching items

FOLD (reduce)

Reduces a whole list to a single value by repeatedly applying a combining function.

fold (+) 0 [2, 3, 2]
=  0 + 2 = 2
=  2 + 3 = 5
=  5 + 2 = 7  →  7

💡 Think: accumulate / collapse to one

💡 Exam technique Always work through fold step-by-step showing the accumulator value at each stage. Don't jump to the answer.

📋 Lists in Functional Programming

4.12.3

Head and Tail

towers = ["Blackpool", "Paris", "New Brighton", "Toronto"]
head(towers)  =  "Blackpool"   ← the FIRST element
tail(towers)  =  ["Paris", "New Brighton", "Toronto"]  ← REST of list
A list  =  head  :  tail       Written as  (x:xs)
[]  =  the EMPTY LIST

List Operations to Know

OperationWhat it returns
head listReturns first element
tail listReturns list minus head
isEmpty listTrue if list is []
length listNumber of elements
prepend x listAdds x to front
append x listAdds x to end

Tracing a Recursive Function (exam favourite!)

total [] = 0
total (x:xs) = x + total (xs)
Evaluate: total [4, 2, 5, 3]
CallExpressionResult
Call 1total [4,2,5,3]4 + total [2,5,3]
Call 2total [2,5,3]2 + total [5,3]
Call 3total [5,3]5 + total [3]
Call 4total [3]3 + total []
Call 5total []0 ← BASE CASE
Unwind3+0=3, 5+3=8, 2+8=10, 4+10=14Result: 14

BASE CASE: function [] = ...

What to do when the list is empty. This STOPS the recursion.

RECURSIVE CASE: function (x:xs) = ...

x is the head (first item). xs is the tail. Function calls itself on the tail — gets smaller each time.

🔒 Immutability & Statelessness

4.11 + 4.12
⚠️ The Examiner's Complaint Students write "immutable" or "stateless" and stop there. That earns ZERO marks. You must DESCRIBE what these mean.

IMMUTABILITY

Data structures / variables cannot be changed after they are created. There are no "variables" in the traditional sense — values are fixed.

  • In Python: x = 5 then x = 10 (x changes)
  • In a functional language: once x = 5 it STAYS 5
  • A "new" value creates a new binding — old one is untouched

📡 Why useful: Multiple processors can safely read the same data without risk of one process changing it for another. Enables safe distributed computing.

STATELESSNESS

A function's output depends ONLY on its inputs. Functions have no "memory" of previous calls — no side-effects.

  • Calling double 4 always returns 8
  • Doesn't matter how many times you've called it before
  • No global variables, no hidden state

📡 Why useful: The order of execution doesn't matter — functions can run in any order or simultaneously. Makes distributed and parallel computing much easier.

💡 Model answer "Immutable means the values stored in data structures cannot be changed once they are created, so multiple processors can safely access the same data."

📝 Worked Exam Question Walkthrough

June 2018 — Q15 
fw [a,b] = a * b
fx c = map fw c
fy d = fold (+) 0 d
fz e = fy (fx e)
sales = [[10,2], [2,25], [4,8]]
Question 1 1 mark

How many of the four functions use a higher-order function?

Answer 2 — fx uses map; fz calls fx which uses map.
Question 2 3 marks

Calculate: fw [4,3], fx sales, fz sales

Step-by-step worked answers

Function callWorkingsResult
fw [4,3]fw [a,b] = a * b → 4 * 312
fx salesmap fw [[10,2],[2,25],[4,8]] → apply fw to each pair
fw[10,2]=20, fw[2,25]=50, fw[4,8]=32		[20, 50, 32]
fz salesfy (fx sales) = fy [20, 50, 32]
fold (+) 0 [20,50,32]: 0+20=20, 20+50=70, 70+32=102		102

Strategy for these questions

  1. Read function definitions carefully — the syntax is always explained in the question.
  2. Work INSIDE OUT: evaluate the innermost function first, then pass its result outward.
  3. For map: apply the function to each list item and write the new list.
  4. For fold: write out each step showing the accumulator — don't jump to the answer.
  5. For "how many use a HOF?": look for map/filter/fold in the body of each function.
