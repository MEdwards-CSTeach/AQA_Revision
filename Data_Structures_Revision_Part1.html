Stacks

LIFO · Push · Pop · Peek · Static & Dynamic

A stack is a Last In, First Out (LIFO) abstract data type. Items are added to and removed from the TOP only.
📚
Real-World Analogies
  • A stack of plates – add & remove from the top
  • Browser back button – each page pushed; back pops
  • Undo function – each action pushed, undo pops
✅
Ideal Use Cases
  • Expression evaluation (RPN calculators)
  • Reversing data / strings
  • Function call management (call stack)
  • Depth-first search (DFS)
  • Balancing brackets / parentheses
⚖️
Pros & Cons

Pros

  • Simple, fast O(1) push & pop
  • Great for tracking state
  • Memory efficient

Cons

  • No random access to elements
  • Stack overflow if too deep (static)
  • Only top element directly accessible

Operations

push(item)

Add item to top. Check isFull() first.

pop()

Remove & return top. Check isEmpty() first.

peek()

Return top WITHOUT removing. top pointer unchanged.

isEmpty()

Returns true when top == -1

isFull()

Returns true when top == maxSize-1

function push(item):
  if top == maxSize - 1 then
    output "Stack full – overflow error"
  else
    top ← top + 1
    stack[top] ← item

function pop():
  if top == -1 then
    output "Stack empty – underflow error"
  else
    item ← stack[top]
    top ← top - 1
    return item

function peek():
  if top == -1 then return "empty"
  return stack[top]

function isEmpty(): return top == -1
function isFull():  return top == maxSize - 1
class Stack:
    def __init__(self, max_size):
        self.data = [None] * max_size
        self.top = -1
        self.max_size = max_size

    def is_empty(self):
        return self.top == -1

    def is_full(self):
        return self.top == self.max_size - 1

    def push(self, item):
        if self.is_full():
            raise Exception("Stack overflow")
        self.top += 1
        self.data[self.top] = item

    def pop(self):
        if self.is_empty():
            raise Exception("Stack underflow")
        item = self.data[self.top]
        self.top -= 1
        return item

    def peek(self):
        if self.is_empty():
            raise Exception("Stack empty")
        return self.data[self.top]
public class Stack {
    private int[] data;
    private int top, maxSize;

    public Stack(int maxSize) {
        this.maxSize = maxSize;
        data = new int[maxSize];
        top = -1;
    }

    public boolean isEmpty() { return top == -1; }
    public boolean isFull()  { return top == maxSize - 1; }

    public void push(int item) {
        if (isFull()) throw new RuntimeException("Overflow");
        data[++top] = item;
    }

    public int pop() {
        if (isEmpty()) throw new RuntimeException("Underflow");
        return data[top--];
    }

    public int peek() {
        if (isEmpty()) throw new RuntimeException("Empty");
        return data[top];
    }
}
💡 Exam Tips — Stacks
  • Always check isEmpty() before pop() or peek()
  • Always check isFull() before push()
  • top pointer INCREASES on push, DECREASES on pop
  • Dynamic stacks (linked lists) cannot overflow
  • To reverse a queue: pop items onto stack, then dequeue from stack back to queue

Linear Queue

FIFO · Front & Rear Pointers · Static Array

A queue is a First In, First Out (FIFO) structure. Items join at the REAR (enqueue) and leave from the FRONT (dequeue). Two pointers: front and rear.
📋
How it Works
  • Items join at REAR (enqueue)
  • Items leave from FRONT (dequeue)
  • Two pointers track front and rear
  • front advances on dequeue — freed space is NOT reused
✅
Good Use Cases
  • Print spoolers (jobs wait their turn)
  • CPU scheduling (round-robin)
  • BFS graph traversal
  • Message queues / event buffers
  • Ticket booking systems
⚖️
Pros & Cons

Pros

  • Simple to understand & implement
  • Fair FIFO ordering guaranteed
  • O(1) enqueue and dequeue

Cons

  • Wasted space after dequeues
  • Reports FULL even with empty slots at start
  • Solution → use a circular queue

class LinearQueue:
    def __init__(self, size):
        self.q = [None] * size
        self.front = -1
        self.rear  = -1
        self.size  = size

    def is_empty(self): return self.front == -1
    def is_full(self):  return self.rear == self.size - 1

    def enqueue(self, item):
        if self.is_full(): raise Exception("Queue full")
        if self.is_empty(): self.front = 0
        self.rear += 1
        self.q[self.rear] = item

    def dequeue(self):
        if self.is_empty(): raise Exception("Queue empty")
        item = self.q[self.front]
        self.q[self.front] = None
        if self.front == self.rear:
            self.front = self.rear = -1
        else:
            self.front += 1
        return item
public class LinearQueue {
    private String[] q;
    private int front, rear, size;

    public LinearQueue(int size) {
        this.size = size;
        q = new String[size];
        front = rear = -1;
    }

    public boolean isEmpty() { return front == -1; }
    public boolean isFull()  { return rear == size - 1; }

    public void enqueue(String item) {
        if (isFull()) throw new RuntimeException("Full");
        if (isEmpty()) front = 0;
        q[++rear] = item;
    }

    public String dequeue() {
        if (isEmpty()) throw new RuntimeException("Empty");
        String item = q[front];
        q[front] = null;
        if (front == rear) front = rear = -1;
        else front++;
        return item;
    }
}

Circular Queue

Solving Wasted Space · Wrap-Around · Modulo Arithmetic

The Key Formula:
rear = (rear + 1) MOD maxSize — on ENQUEUE
front = (front + 1) MOD maxSize — on DEQUEUE

This creates a ring — when rear reaches the end it wraps back to index 0, reusing freed space.
🔴
The Linear Queue Problem

After repeated enqueue/dequeue, rear advances to the end and the queue is reported FULL — even though empty slots exist at the front. Space is permanently wasted.

✅
The Circular Solution

The array acts as a ring. Rear wraps around using modulo. Freed space at the front is automatically reused when rear wraps back to those indices.

🧮
Full vs Empty — Critical!

⚠️ You cannot use front==rear alone — this is true for both states!

Use a size counter:
isEmpty: size == 0
isFull: size == maxSize

class CircularQueue:
    def __init__(self, size):
        self.q = [None] * size
        self.max   = size
        self.front = 0
        self.rear  = 0
        self.size  = 0

    def is_empty(self): return self.size == 0
    def is_full(self):  return self.size == self.max

    def enqueue(self, item):
        if self.is_full(): raise Exception("Full")
        self.q[self.rear] = item
        self.rear = (self.rear + 1) % self.max   # WRAP!
        self.size += 1

    def dequeue(self):
        if self.is_empty(): raise Exception("Empty")
        item = self.q[self.front]
        self.q[self.front] = None
        self.front = (self.front + 1) % self.max  # WRAP!
        self.size -= 1
        return item
public class CircularQueue {
    private String[] q;
    private int front, rear, size, max;

    public CircularQueue(int max) {
        this.max = max;
        q = new String[max];
        front = rear = size = 0;
    }

    public boolean isEmpty() { return size == 0; }
    public boolean isFull()  { return size == max; }

    public void enqueue(String item) {
        if (isFull()) throw new RuntimeException("Full");
        q[rear] = item;
        rear = (rear + 1) % max;  // WRAP AROUND!
        size++;
    }

    public String dequeue() {
        if (isEmpty()) throw new RuntimeException("Empty");
        String item = q[front];
        q[front] = null;
        front = (front + 1) % max;  // WRAP AROUND!
        size--;
        return item;
    }
}

Priority Queue

Items Ordered by Priority · Not by Arrival Time

In a priority queue, items are dequeued in priority order, not arrival order. The item with the highest priority is always removed first. Items with equal priority are served FIFO.
📋
How it Works
  • Each item has a priority value
  • On dequeue: remove highest priority item
  • Implementation: sorted array (O(n) insert, O(1) remove)
  • Or: unsorted array (O(1) insert, O(n) remove)
✅
Ideal Use Cases
  • A&E hospital triage (critical patients first)
  • OS process scheduling (real-time tasks)
  • Network packet routing (QoS)
  • Dijkstra's shortest path algorithm
  • Print queues with job priorities
⚖️
Pros & Cons

Pros

  • Critical items processed first
  • Flexible ordering

Cons

  • More complex than FIFO queue
  • Low-priority items may starve
  • Insertion O(n) for sorted array

class PriorityQueue:
    def __init__(self):
        self.q = []  # list of (priority, item) tuples

    def enqueue(self, item, priority):
        # Insert in sorted position (descending priority)
        pos = 0
        while pos < len(self.q) and self.q[pos][0] >= priority:
            pos += 1
        self.q.insert(pos, (priority, item))

    def dequeue(self):
        if not self.q: raise Exception("Empty")
        return self.q.pop(0)[1]  # remove from front

    def is_empty(self): return len(self.q) == 0

Hash Maps & Hash Tables

Key-Value Pairs · Hashing · Collisions · Rehashing

A hash table maps keys to values using a hash function to compute an array index, enabling near O(1) average-case lookup, insertion and deletion.
⚙️
The Hashing Process
  • Step 1: Take a key (e.g. "Alice")
  • Step 2: Apply hash function → index
  • Step 3: Store value at that index
  • Lookup: hash again → go directly → O(1)!
✅
Ideal Use Cases
  • Dictionaries & symbol tables
  • Database indexing
  • Caching (web / CPU)
  • Counting word frequency
  • Password storage (cryptographic hash)
⚖️
Pros & Cons

Pros

  • Average O(1) lookup, insert, delete
  • Very efficient for large datasets
  • Flexible key types

Cons

  • Collisions degrade performance
  • Worst case O(n) with many collisions
  • No ordering of keys

class HashMap:
    def __init__(self, size=7):
        self.size = size
        self.keys = [None] * size
        self.vals = [None] * size

    def _hash(self, key):
        return sum(ord(c) for c in str(key)) % self.size

    def put(self, key, value):
        idx = self._hash(key)
        while self.keys[idx] is not None:
            if self.keys[idx] == key:   # update existing
                self.vals[idx] = value
                return
            idx = (idx + 1) % self.size  # linear probe
        self.keys[idx] = key
        self.vals[idx] = value

    def get(self, key):
        idx = self._hash(key)
        while self.keys[idx] is not None:
            if self.keys[idx] == key:
                return self.vals[idx]
            idx = (idx + 1) % self.size
        return None  # not found
public class HashMap {
    private String[] keys, vals;
    private int size;

    public HashMap(int size) {
        this.size = size;
        keys = new String[size];
        vals = new String[size];
    }

    private int hash(String key) {
        int sum = 0;
        for (char c : key.toCharArray()) sum += (int) c;
        return sum % size;
    }

    public void put(String k, String v) {
        int idx = hash(k);
        while (keys[idx] != null) {
            if (keys[idx].equals(k)) { vals[idx] = v; return; }
            idx = (idx + 1) % size;
        }
        keys[idx] = k; vals[idx] = v;
    }

    public String get(String k) {
        int idx = hash(k);
        while (keys[idx] != null) {
            if (keys[idx].equals(k)) return vals[idx];
            idx = (idx + 1) % size;
        }
        return null;
    }
}
💡 Exam Tips — Hash Maps
  • Know how to apply a hash function manually (e.g. ASCII sum MOD tableSize)
  • A collision = two different keys produce the same hash value
  • Rehashing = finding the next available slot (linear probing)
  • The hash function determines WHERE data is stored
  • A dictionary is a key-value pair ADT — typically implemented using a hash table
