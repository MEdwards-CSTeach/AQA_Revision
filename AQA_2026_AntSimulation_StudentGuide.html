System Overview

What is the Ant Simulation?

An ant colony simulation played out on a 2D grid. Each cell can contain food, an ant nest, ants and pheromone trails. The simulation advances stage by stage — at each stage ants move, pick up or deposit food, and lay pheromone trails. Nests consume food each stage and may spawn or lose ants depending on food levels.

The user can choose from four pre-set simulations, then inspect the grid or advance the simulation through a menu.

Key Concepts from the Preliminary Material

  • Queen ants stay in their nest and reproduce
  • Worker ants move around searching for food
  • Workers carrying food return directly to nest, laying pheromones
  • Workers with no food follow strongest pheromone nearby, or move randomly
  • Pheromones decay each stage; removed when strength = 0
  • Nests with low food lose ants randomly

Menu Options

  • 1 – Display overall details (all cells)
  • 2 – Display area details (start row/col → end row/col)
  • 3 – Inspect a single cell in detail
  • 4 – Advance one stage
  • 5 – Advance X stages
  • 9 – Quit

Classes & Methods — Detailed Reference

Entity Base Class abstract behaviour

The root of the object hierarchy. Holds a grid position (row, column) and an id. All objects that live on the grid extend Entity. The advanceStage and getDetails methods are defined but empty here — subclasses override them to provide meaningful behaviour (this is the Template Method / polymorphism pattern).

getColumn()—intReturns current column.
getID()—intReturns the unique id.
advanceStage(nests, ants, pheromones)ArrayListsvoidEmpty in base class. Overridden by Ant, Nest and Pheromone to implement their stage logic.
getDetails()—StringReturns "" in base class. Overridden by Cell, Ant and WorkerAnt to build display strings.

Cell extends Entity

Represents one square on the 2D grid. Stores how much food is present. Cells are created for every (row, column) combination at startup. Cells themselves don't move — they're a static container.

MethodParametersReturnsPurpose
Cell(startRow, startColumn)int, int—Constructor. Creates cell at position; food starts at 0.
getAmountOfFood()—intReturns how much food is in this cell.
updateFoodInCell(change)intvoidAdds change to amountOfFood. Pass negative value to reduce. No lower-bound check here.
getDetails() @Override—StringReturns food amount as a display string.

Ant extends Entity superclass for ants

Base ant class. Tracks which nest the ant belongs to (nestRow, nestColumn), how much food it's carrying, how many stages it's been alive, and its food capacity. chooseCellToMoveTo is defined but empty here — WorkerAnt overrides it with real movement logic. QueenAnt does not override it (queens don't move).

Uses a static counter nextAntID to give each ant a unique ID across the whole simulation.

QueenAnt extends Ant

Sets typeOfAnt = "queen". Does not override any methods — queens never move (foodCapacity stays 0, chooseCellToMoveTo stays empty). The queen's role is reproduced by the Nest.advanceStage method, not by the queen ant object itself.

WorkerAnt extends Ant

Sets typeOfAnt = "worker" and foodCapacity = 30. Overrides two methods. The chooseCellToMoveTo override is the core movement algorithm.

MethodReturnsPurpose

chooseCellToMoveTo(listOfNeighbours, indexOfStrongestPheromone) @Overridevoid Three-way decision:

2. If no food & no pheromone detected: choose a random valid neighbour.
3. If no food & pheromone detected: move to the neighbour with the strongest pheromone.

Tracks food reserves and how many queens are present. Has a static nextNestID counter (like ants). The advanceStage override contains the most complex logic in the program.

Complex Algorithms Explained
⚠️
Exam focus: AQA always asks students to trace, explain or modify algorithms. These are the ones most likely to appear.

1 — Grid Index Calculation: getIndex(row, column)

The 2D grid is stored as a flat (1D) ArrayList. This formula converts a (row, column) pair into the correct list index.

private int getIndex(int row, int column) {
    return (row - 1) * numberOfColumns + column - 1;
}

Why row - 1 and column - 1? Because rows and columns are 1-indexed (start at 1) but ArrayList positions are 0-indexed (start at 0).

Example: 5-column grid, get cell at row 3, column 2

(3 - 1) × 5 + (2 - 1) = 2 × 5 + 1 = 11

So the cell is at index 11 in the ArrayList — which is the 12th element (0-based).

2 — Finding Neighbours: getIndicesOfNeighbours(row, column)

Finds the (up to 8) neighbouring cells of a given cell by trying all combinations of row and column offsets of {−1, 0, +1}. The combination (0, 0) is skipped (that's the cell itself).

for (rowDirection : {-1, 0, 1}) {
    for (columnDirection : {-1, 0, 1}) {
        if (rowDirection != 0 || columnDirection != 0) {
            // check if neighbourRow and neighbourColumn are in bounds
            // if yes: add getIndex(neighbourRow, neighbourColumn)
            // if no: add -1  (signals "off the grid")
        }
    }
}
ℹ️
Important: The returned list always has 8 slots. Off-grid positions get -1. Code that uses this list always checks if (index != -1) before accessing the grid.

Corner cells (e.g. row 1, col 1) will have 3 valid neighbours and 5 entries of -1. Edge cells will have 5 valid neighbours.

3 — Worker Ant Movement: WorkerAnt.chooseCellToMoveTo

This is the most important algorithm in the program. Three-branch decision:

1
Carrying food (amountOfFoodCarried > 0): Move directly towards home nest — reduce the difference in row and column by 1 each stage. No random element. Note: this is Manhattan-distance navigation, NOT diagonal movement.
2
Empty-handed, no pheromone nearby (indexOfNeighbourWithStrongestPheromone == -1): Choose a random valid neighbour using chooseRandomNeighbour, then call changeCell to update position.
3
Empty-handed, pheromone detected: Find the index within listOfNeighbours that corresponds to the strongest pheromone cell, then call changeCell to move there.
// Branch 1: returning to nest with food
if (amountOfFoodCarried > 0) {
    if (row > nestRow)    row -= 1;
    else if (row < nestRow) row += 1;
    if (column > nestColumn)    column -= 1;
    else if (column < nestColumn) column += 1;
}
// Branch 2: random wander
else if (indexOfNeighbourWithStrongestPheromone == -1) {
    int indexToUse = chooseRandomNeighbour(listOfNeighbours);
    changeCell(indexToUse, r, c);
}
// Branch 3: follow pheromone
else {
    int indexToUse = listOfNeighbours.indexOf(indexOfNeighbourWithStrongestPheromone);
    changeCell(indexToUse, r, c);
}
⚠️
Key observation — IntWrapper: Java passes objects by reference but primitives by value. To allow changeCell to modify row and column, a helper class IntWrapper is used — it wraps an int in an object so its value can be changed inside the method and seen outside. This is a common pattern when Java's pass-by-value would otherwise prevent modification.

4 — Cell Index to Direction: changeCell(newCellIndicator, row, col)

Converts a neighbour index (0–8, representing a position in the 3×3 grid of neighbours) into an actual row/column change. The mapping:

Index01234 (self—skipped)5678
Row change−1−1−1000+1+1+1
Col change−10+1−10+1−10+1

So: indices 0,1,2 → row decreases (go up); 6,7,8 → row increases (go down); 0,3,6 → column decreases (go left); 2,5,8 → column increases (go right).

5 — Nest Food & Population: Nest.advanceStage

Called once per stage per nest. Handles feeding, starvation, culling and reproduction.

1
Calculate food cost: For each ant belonging to this nest: queens cost 10 food, workers cost 2 food. Call changeFood(-count).
2
Calculate ants to cull (antsToCull): starts at 0.
— If food == 0 and there are workers: add 1
— If food < number of workers: add 1
— If food < workers × 5: add 1 (but cap at total workers)
3
Culling: If antsToCull > 0, randomly remove that many ants belonging to this nest. If a culled ant is a queen, decrement numberOfQueens.
4
Reproduction (else branch — only if food ≥ workers × 5): For each queen: 50% chance (rNo1 < 50) she reproduces. If reproducing: 2% chance (rNo2 < 2) of new QueenAnt, 98% chance of new WorkerAnt.
⚠️
Potential bug: The culling block is only entered when food < workers × 5, but two of the three antsToCull increments are checked outside this condition. Ants could be culled when food > 0 in some edge cases. The exam paper notes "there could be errors in the implementation."

6 — Main Stage Loop: Simulation.advanceStage

Processes one complete simulation stage. Order matters:

1
Decay all pheromones. Any pheromone with strength = 0 is added to a delete list, then removed after the loop.
2
Advance each ant (increments stages counter).
3
Per ant — three-way action check:
a. At own nest & carrying food → deposit food to nest (addFoodToNest), clear food carried.
b. Cell has food & ant empty & ant can carry food → pick up a random valid amount from the cell.
c. Otherwise → if carrying food, lay pheromone; then call chooseCellToMoveTo to move.
4
Advance each nest (feeding + reproduction/culling).

Tier 1 — Small amendments (validation / tracking)

4–5 marks

Add input validation so that when the user enters a number of stages to advance by, it must be between 1 and 100 (inclusive); repeat until a valid value is entered.
Validation / Tracking
Hint: Add a stages counter variable. Use a do-while loop around the input. Display an error message for invalid inputs.
4–5 marks
Modify the menu in main so that menu option 3 (Inspect cell) validates the row and column entered by the user — they must be within the grid bounds. If an invalid value is entered, display an error message and ask again.
Validation
Hint: Wrap the input in a loop. Check row >= 1 && row <= numberOfRows. The Simulation object doesn't expose its grid size directly — think about how to access numberOfRows.
5–6 marks
Modify the program to track and display the total amount of food collected by all worker ants across the entire simulation. This should be displayed when the user inspects overall details.
Tracking
Hint: Add a totalFoodCollected attribute to Simulation. Increment it in advanceStage when food is picked up (foodObtained). Include it in getDetails().

Tier 2 — New method / extended feature (~9–12 marks)

9–11 marks

10–12 marks

10–12 marks

New Method

Tier 3 — New subclass (~12–14 marks)

13–14 marks
Create a new class ScoutAnt that extends WorkerAnt. A ScoutAnt has a food capacity of 0 (it never picks up food). Instead of following pheromones, it always moves randomly. Theory questions about the skeleton code (Section C style)

1 mark
State the purpose of the IntWrapper class.
OOP
Answer: Java passes primitive types (like int) by value, not by reference, so a method cannot modify the caller's variable. IntWrapper wraps an int in an object so its value field can be changed inside a method and the change is visible to the caller.
2 marks
Explain why the QueenAnt class does not override chooseCellToMoveTo.
OOP
Answer: Queens do not move — they stay in their nest. The base class Ant's chooseCellToMoveTo is empty (does nothing), which is the correct behaviour for a queen. There is therefore no need to override it.
1 mark
State what value is stored in listOfNeighbours for a neighbour that is off the grid.
Trace
Answer: -1
2 marks
Explain what polymorphism means and give an example of it in the skeleton program.
OOP Theory
Answer: Polymorphism means different objects respond differently to the same method call. Example: when advanceStage is called on an element in the ants ArrayList, a WorkerAnt executes its overriding version (which also handles food/movement), while a QueenAnt executes the Ant base class version (which only increments stages).
2 marks
A pheromone can be reinforced rather than duplicated. Describe how the code achieves this.
Algorithm
Answer: In updateAntsPheromoneInCell, the pheromones list is searched for one that is at the same location AND belongs to the same ant (matching ant ID using getBelongsTo). If found, its strength is increased using updateStrength. Only if no such pheromone is found is a new Pheromone object created and added to the list.
3 marks
Trace the formula getIndex(row, column) for a 5-column grid. Show the index returned for: (a) row 1, col 1   (b) row 2, col 3   (c) row 5, col 5.
Trace
Answers: (a) (1-1)×5 + (1-1) = 0  |  (b) (2-1)×5 + (3-1) = 7  |  (c) (5-1)×5 + (5-1) = 24
3 marks
Explain the three conditions that cause worker ants to be culled in Nest.advanceStage.
Algorithm
Answer: (1) If food reaches 0 AND there are workers — 1 ant culled. (2) If food < number of workers — 1 more ant culled. (3) If food < workers × 5 — 1 more culled (but capped at total workers). The ants are chosen randomly from those belonging to that nest.
