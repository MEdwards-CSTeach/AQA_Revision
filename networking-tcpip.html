Network Fundamentals & Packet Switching
Packets · Routers · Checksums · Network Topologies
⏱ 15 minutes
Header
Source IP · Dest IP · Packet number · Protocol
Payload (Data)
The actual data being transmitted
Trailer
Checksum for error detection

📦 Why Packets?

  • Large files are split into small packets for transmission
  • Each packet may travel via a DIFFERENT route across the internet
  • Reassembled at the destination using sequence numbers
  • Allows efficient use of network resources and fault tolerance

🔀 Router's Role in Packet Switching

  • Adds source/destination IP addresses to packets
  • Selects the best next 'hop' using routing tables
  • Performs error checking on packet headers
  • Forwards each packet independently

✅ Checksum

  • A mathematical value calculated from the packet data
  • Receiver recalculates checksum and compares to trailer value
  • If they differ → packet is corrupted → discard/request again

🌐 Network Types

  • LAN — single site, privately owned
  • WAN — multiple sites, large geographic area
  • Topologies: Star, Bus, Mesh, Ring — know diagrams and advantages

The TCP/IP Stack & Application Protocols
Four Layers · What Each Adds · Ports · FTP / HTTP / SMTP / POP3 / SSH
⏱ 25 minutes
Layer 4
Application
Formats data using application protocol (HTTP, SMTP, FTP etc). Uses well-known ports: HTTP=80, HTTPS=443, SMTP=25, POP3=110, FTP=21, SSH=22
Layer 3
Transport
Splits data into segments, adds source & destination port numbers, sequence numbers, checksum. Handles retransmission. TCP (reliable, connection-oriented) or UDP (fast, connectionless)
Layer 2
Network
Adds source & destination IP addresses, encapsulates into datagram, adds Time to Live (TTL). Uses subnet mask to determine if destination is on same subnet.
Layer 1
Link
Adds hardware (MAC) address of destination / next router / gateway. Uses device drivers / NIC. Puts data onto the physical medium (cable, WiFi signal).

Application Layer Protocols

ProtocolPortPurpose
HTTP80Transfer web pages (text) from server to browser. Stateless – each request is independent.
HTTPS443Secure HTTP using TLS/SSL encryption. Used for banking, login pages, sensitive data.
FTP21Transfer files between client and server. Supports anonymous and authenticated access.
SMTP25Send emails from client to email server, or between email servers.
POP3110Retrieve emails from server to client. Downloads and (usually) deletes from server.
SSH22Secure encrypted remote login. Used for remote management — executing commands on servers securely.

🔌 Ports

  • Well-Known (0–1023): reserved for standardised services, always the same (e.g. HTTP always port 80)
  • Client / Ephemeral (1024–65535): randomly assigned to client when making a connection; distinguishes multiple simultaneous connections
  • Socket: IP address + port number, e.g. 192.168.1.10:443 — uniquely identifies one end of a TCP connection

🌐 WebSocket Protocol

  • Persistent, full-duplex connection between browser and server over TCP
  • Unlike HTTP (request/response), either side can send data at any time
  • Used in real-time apps: live chat, multiplayer games
REST API: GET→SELECT · POST→INSERT · PUT→UPDATE · DELETE→DELETE

IP Addressing, NAT & DHCP
IPv4 & IPv6 · Subnet Masking · Public/Private · NAT · DHCP · Port Forwarding
⏱ 25 minutes
IPv4 — 32 bits
  • 4 octets in decimal: 192.168.1.100
  • ~4.3 billion unique addresses (not enough!)
  • Private ranges: 192.168.x.x / 10.x.x.x / 172.16–31.x.x
  • Private = non-routable (can't reach internet directly)
  • x.x.x.0 = Network address; x.x.x.255 = Broadcast
IPv6 — 128 bits
  • 8 groups of 4 hex digits: 2001:0db8:…
  • 340 undecillion unique addresses
  • Introduced because IPv4 address space was exhausted
  • Every device can have a unique, routable, public IP
  • No longer need NAT as a workaround

Subnet Masking — Worked Example

# Given: 27 bits allocated to network ID
Binary mask: 11111111.11111111.11111111.11100000
Dotted decimal: 255.255.255.224

# Calculate usable hosts
Host bits = 32 − 27 = 5 bits
Total addresses = 2⁵ = 32
Usable hosts = 32 − 2 = 30 (minus network & broadcast)

# Find network address: AND the IP with the mask
IP: 192.168.1.100 → 11000000.10101000.00000001.01100100
Mask: 255.255.255.224 → 11111111.11111111.11111111.11100000
AND result → 192.168.1.96 (network address)

NAT — Network Address Translation

① Computer on LAN (192.168.1.5) sends request to internet
② NAT router REPLACES private source IP with its own public IP (e.g. 82.34.210.1) and generates a new port number
③ Router stores mapping: private IP + original port ↔ public port in its Translation Table
④ When reply arrives, router looks up mapping, restores private IP, forwards to correct device
Private IPPrivate Port↔Public IPPublic Port
192.168.1.53456↔82.34.210.151200
192.168.1.74892↔82.34.210.151201
WHY? Allows many devices with private IPs to share ONE public IP address — helps conserve IPv4 addresses.

DHCP — Four-Step Handshake

Step 1
DISCOVER
Host broadcasts to find a DHCP server
Step 2
OFFER
DHCP server offers an IP address configuration
Step 3
REQUEST
Host accepts one offer (echoes it back)
Step 4
ACKNOWLEDGE
Server confirms the lease
Why DHCP? Automates configuration, avoids duplicate IPs, reuses addresses when devices leave the network.

↪ Port Forwarding

  • Allows external traffic arriving on a specific port to be directed to a specific internal IP
  • E.g. traffic arriving on port 80 (HTTP) from outside is forwarded to the web server at internal IP 192.168.1.12
  • Without it, NAT blocks all unsolicited incoming traffic

Mark Scheme

Q1 — Checksum [2]
  • Checksum used for error detection — checks if packet has been corrupted in transit
  • Receiver recalculates checksum from received data and compares with the one in the packet; if different, packet is discarded / re-requested
Q2 — Transport Layer [3]
  • Splits data into segments
  • Adds source and destination port numbers to each segment
  • Adds checksum / sequence number to segment; handles retransmission of lost or corrupted segments
Q3 — Subnet Mask [2]
  • (a) 25 bits → 11111111.11111111.11111111.10000000 → 255.255.255.128
  • (b) Host bits = 32−25 = 7 → 2⁷ = 128 addresses → 126 usable hosts (−2 for network and broadcast)
Q4 — NAT [4]
  • NAT router replaces private (source) IP with its own public IP when request leaves the LAN
  • Router also replaces source port with a generated port number and stores the mapping in a translation table
  • When reply arrives, port number is looked up, private IP is restored, packet forwarded to correct device
  • Necessary because not enough IPv4 addresses for every device to have a unique public IP

Common Exam Mistakes — Don't Lose Easy Marks

✗
Confusing 'routing' with what the Network Layer adds.
The Network Layer ADDS source and destination IP addresses. The mark scheme rejects just saying 'routes data' — you must say it adds the IP addresses.
✗
Saying NAT 'gives devices an IP address'.
NAT replaces the private source IP with the router's public IP. DHCP gives devices IP addresses. These are completely different mechanisms.
✗
Subnet mask calculation errors.
Count 1-bits carefully. 27 bits = 11111111.11111111.11111111.11100000. Host bits (5) give 2⁵=32 addresses, minus 2 = 30 usable hosts.
✗
Using 'packet' where the mark scheme says 'segment' or 'datagram'.
Segment = Transport layer. Datagram = Network layer. 'Packet' is sometimes accepted but can lose marks.
✗
DHCP — missing steps of the handshake.
You must state Discover → Offer → Request → Acknowledge. Saying 'the server gives the device an IP' only earns 1 mark maximum.
✗
IPv6 reason — too vague.
'More addresses' alone is rejected. You must say: 'enough addresses so every device can have a unique, globally routable (public) IP address'.
