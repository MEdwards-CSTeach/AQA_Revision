Definition

A Turing machine is a theoretical abstract machine that manipulates symbols on an infinite tape according to a fixed set of rules. It provides a formal model of what it means for a function to be computable.

Think of it as a computer with one fixed program.

Components — TASH

📼
T
Tape
Infinite strip of cells, each holding one symbol. Read/write memory AND input AND output all in one.
🔤
A
Alphabet (Σ)
Finite set of symbols the machine can read or write. Must include a blank symbol (shown as _ or □).
⚙️
S
States
Set of states the machine can be in, including one start state and at least one halt state.
🔎
H
Head (Read/Write)
Reads the current cell, writes a new symbol, then moves left or right one cell.
🧠 TASH: Tape · Alphabet · States · Head — the four things that define a Turing machine.

⚠️ Common Misconception

Students often think the tape is just a memory store like RAM. It is NOT. The tape is the input AND the working memory AND the output all combined.

States & Transition Rules

Transition Function δ

δ(current state, read symbol)
 = (new state, write symbol, direction)

δ(q0, 0) = (q1, 1, R)
δ(q1, 1) = (qH, 1, R)
R = move right  |  L = move left
No outgoing transition = HALT

Halting State

A state with NO outgoing transitions. The machine stops here. This is normal successful termination — NOT an error or crash.

🧠 δ = 'Do': δ tells the machine what to DO next — (state, write, direction).

Why Turing Machines?

Purpose of a Turing Machine

  • Provide a formal mathematical model of computation
  • Define precisely what 'computable' means
  • Prove that some problems CANNOT be solved by any computer
  • Give a theoretical foundation for all of computer science

Universal Turing Machine (UTM)

  • A UTM can simulate ANY other Turing machine
  • The target TM is described on the UTM's tape
  • This is the theoretical blueprint for the stored-program computer
  • One general machine can do everything a specialised machine can

Turing Machine vs Real Computer

FeatureTuring MachineReal Computer
MemoryInfinite tapeFinite RAM/storage
ProgramFixed, single programMany programs loaded from storage

⚠️A Turing machine is NOT faster or more powerful than a real computer. It is simpler and theoretical. A UTM IS a Turing machine — it cannot compute more than a standard TM.

Computable vs Non-Computable Problems

✅ Computable Problems
  • Can be solved by a TM that always halts
  • Examples: sorting, searching, arithmetic, string manipulation
  • Equivalent to: can be solved by any modern computer given enough time/memory
❌ Non-Computable Problems
  • No Turing machine can solve them for all possible inputs
  • UNDECIDABLE — not just slow, genuinely unsolvable
  • Examples: Halting Problem, Busy Beaver problem
  • Even with infinite time and memory — impossible

Church-Turing Thesis

"Any function that can be computed at all, can be computed by a Turing machine." — The definition of computability itself.

🧠 Non-computable ≠ difficult. It means IMPOSSIBLE. If a TM can't solve it, NO computer can — ever, regardless of speed or memory.

The Halting Problem

Definition — Learn This Precisely

"The Halting Problem is the problem of determining, from a description of any computer program and its input, whether the program will eventually halt (stop running) or continue to run forever. Alan Turing proved in 1936 that no algorithm can solve this for all possible program-input pairs."

Why Does It Matter?

  • Proves computers have fundamental, inescapable limits
  • No anti-virus/debugger can perfectly detect all infinite loops
  • Impacts program verification — we cannot write a perfect checker
  • Foundation of computability theory in CS

Significance for Computation

  • Shows some problems are UNDECIDABLE, not just hard
  • The most famous non-computable problem
  • Turing used a TM to frame and prove this
  • Every real computer is bounded by what TMs can do

Proof: The Halting Problem is Unsolvable

Proof by contradiction — understand the argument, not every detail.

1
Assume a solver exists

Suppose there exists a program H(P, I) that returns "halts" if program P with input I halts, or "loops" if it runs forever.

2
Build a paradox machine D

Create program D that uses H: if H says P halts when given itself as input, D loops forever. If H says P loops, D halts.

3
Run D on itself

Ask: what does D(D) do?
If D halts → H said D halts, so D must loop. Contradiction!
If D loops → H said D loops, so D must halt. Contradiction!

4
Conclusion

Either outcome is a contradiction. Therefore, our assumption is wrong — no such solver H can exist. The Halting Problem is UNDECIDABLE.

🧠 ACRC: Assume · Construct · Run · Contradiction — the four steps of the halting problem proof.

Common Misconceptions

✗ Wrong
A Turing machine is an actual physical device.
✓ Correct
It is a theoretical/abstract model — it has never been physically built in its infinite form.
✗ Wrong
Non-computable means too hard or too slow.
✓ Correct
Non-computable means IMPOSSIBLE. No computer, however fast, can ever solve it for all inputs.
✗ Wrong
The Halting Problem just asks if a specific program stops.
✓ Correct
It asks if a GENERAL algorithm can decide this for ANY program with ANY input.
✗ Wrong
The halt state is like a crash or error.
✓ Correct
Halting is normal termination — the machine completes successfully and stops.
✗ Wrong
A UTM is more powerful than a Turing machine.
✓ Correct
A UTM IS a Turing machine. It can simulate others but cannot compute more than a TM.

Exam Practice Questions

[2 marks]
State what is meant by a 'halting state' in a Turing machine.
State with no outgoing transitions; machine stops.
[3 marks]
Explain, using an example, the difference between a computable and a non-computable problem.
Computable = TM halts with answer (e.g. sorting); Non-computable = no TM can solve for all inputs (e.g. Halting Problem).
[4 marks]
Describe the Halting Problem and explain its significance for computation.
Define HP; state Turing proved no algorithm can solve it; conclude some problems fundamentally cannot be solved by computers.
[2 marks]
Give two components of a Turing machine and describe the role of each.
Any 2 of: tape (read/write memory); read-write head (reads/writes, moves); states (track computation phase); alphabet (valid symbols).
📝'Describe' and 'Explain' questions want WHY, not just WHAT. Always link to consequences.
