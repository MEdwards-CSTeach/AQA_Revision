Signed Binary – Two's Complement

Why Two's Complement?

Unsigned binary can't represent negative numbers. Two's complement treats the MSB (most significant bit) as a negative weight and is used by every modern CPU because:

  • Only ONE representation of zero (sign-and-magnitude has +0 and −0 — a major headache)
  • Subtraction uses the same hardware adder as addition — no separate subtract circuit
  • Overflow detection is straightforward

Bit Weights in 8-bit Two's Complement

Bit 7 (MSB)Bit 6Bit 5Bit 4Bit 3Bit 2Bit 1Bit 0
−128 +64+32+16 +8+4+2+1
Range formula: For n bits →  −2n−1  to  +2n−1 − 1
8-bit: −128 to +127    16-bit: −32,768 to +32,767

Representing a Positive Number

Positive numbers have MSB = 0. Convert to binary normally — it's already in two's complement form.

Example: Represent +45 in 8-bit two's complement

Step 1
Convert 45 to binary: 32 + 8 + 4 + 1 = 00101101
Step 2
MSB = 0 ✓ confirms positive — no further action needed
Verify
−128×0 + 64×0 + 32×1 + 16×0 + 8×1 + 4×1 + 2×0 + 1×1 = 45 ✓
Answer
00101101

Representing a Negative Number – Flip and Add 1

To represent a negative number:

Step 1
Write the magnitude as n-bit unsigned binary
Step 2
Flip (invert) ALL bits — every 0→1 and every 1→0
Step 3
Add 1 to the result
Check
MSB should now be 1 (confirming negative)

Example: Represent −45 in 8-bit two's complement

Step 1
Write +45: 00101101
Step 2
Flip bits: 11010010
Step 3
Add 1: 11010010 + 00000001 = 11010011
Verify
−128 + 64 + 16 + 2 + 1 = −128 + 83 = −45 ✓
Answer
11010011

Subtraction Using Two's Complement

A − B = A + (−B)  — convert B to its two's complement negative, then add normally.

Example: 18 − 72 in 8-bit two's complement

Step 1
+18 in binary: 00010010
Step 2
+72 in binary: 01001000
Step 3
Find −72: flip: 10110111
Step 4
Add 1 → −72: 10111000
Step 5
Add 18 + (−72): 00010010 + 10111000 = 10101010 ← carry out discarded
Step 6
MSB = 1 → negative. Find magnitude: flip 10101010 → 01010101, add 1 → 01010110 = 54
Answer
−54 (18 − 72 = −54 ✓)
🔑 Key Exam Tips – Two's Complement
  • Always flip ALL 8 bits — a common error is flipping only the 1-bits
  • Adding two positives that gives a negative = overflow (not −1 + 1 = 0!)
  • You can also check: add using bit weights directly — MSB has weight −128
  • The "flip and add 1" method works in BOTH directions (positive→negative and negative→positive)

Overflow – Why Leading Bits Are Dropped

What is Overflow?

Overflow occurs when a calculation produces a result that is too large or too small to fit in the register. The result appears incorrect.

Why are leading bits dropped?
A CPU register has a fixed physical bit-width (e.g. 8 bits). There is literally no hardware storage for a 9th bit. When a carry is generated out of the MSB, it has nowhere to go — it is silently lost. This is a hardware constraint, not a software bug.

Overflow Example: 100 + 100 in 8-bit

Step 1
+100 in binary: 01100100
Step 2
+100 in binary: 01100100
Step 3
01100100 + 01100100 = 11001000 ← MSB = 1, reads as −56 in two's complement!
Why?
True result = 200, which requires 9 bits (11001000 + carry). The 9th bit is physically discarded by the fixed-width register.
Result
Overflow! Stored result is −56 instead of +200

Detecting Overflow

  • Adding two positive numbers gives a negative result → overflow
  • Adding two negative numbers gives a positive result → overflow
  • Modern processors set an Overflow Flag (V-flag) that software must check
  • Adding a positive and a negative can never overflow

Fixed Point Binary

What is Fixed Point?

A binary point is placed at a fixed position. Bits to the LEFT are integers; bits to the RIGHT are fractions: ½, ¼, ⅛, 1/16 …

8421●½¼⅛1/16
8421 . 0.50.250.1250.0625

Decimal to Fixed Point – The Multiply-by-2 Method

For the fractional part: repeatedly multiply by 2. The integer part of each result is the next binary digit (read top to bottom).

Example: Convert 6.625 to fixed point (4 integer, 4 fractional bits)

Step 1
Integer: 6 = 4+2 → 0110
Step 2
0.625 × 2 = 1.25 → bit = 1, carry 0.25
Step 3
0.25 × 2 = 0.5 → bit = 0, carry 0.5
Step 4
0.5 × 2 = 1.0 → bit = 1, carry 0 (stop)
Verify
0110.1010 → 4+2 + ½+⅛ = 6 + 0.625 = 6.625 ✓
Answer
0110.1010

When Fractions Cannot Be Represented Exactly

0.1 in decimal = 0.000110011001100… (infinite) in binary.
It can never be stored exactly in any finite number of bits. The value is always rounded → rounding error. This is why 0.1 + 0.2 ≠ 0.3 exactly in most programming languages.

Benefits & Limitations of Fixed Point

✓ Benefits

  • Faster arithmetic — simpler hardware
  • Predictable precision throughout range
  • No complex exponent logic
  • Used in: audio, embedded, real-time systems

✗ Limitations

  • Cannot represent very large AND very small numbers simultaneously
  • Range is fixed by bit allocation
  • Changing precision requires redesign

Floating Point – Mantissa & Exponent

Structure

Value = Mantissa × 2Exponent
Both mantissa and exponent are stored in two's complement
PartStoresMore bits →
MantissaSignificant digitsGreater PRECISION (smaller gaps between representable numbers)
ExponentPower of 2 (scale)Greater RANGE (larger and smaller numbers representable)

Normalisation

A normalised value has maximum precision — no wasted leading bits.

Positive mantissa: must start 01… (MSB=0, next bit=1)
Negative mantissa: must start 10… (MSB=1, next bit=0)

To normalise: shift the binary point until the rule is met, adjust the exponent to compensate.

Positive Decimal → Floating Point

Example: Convert +5.5 to normalised FP (8-bit mantissa, 4-bit exponent)

Step 1
5.5 in binary: 5 = 101, 0.5 = .1 → 101.1
Step 2
Normalise: move binary point left 3 → 0.1011 × 2³
Step 3
Starts 01... ✓ (positive normalised)
Step 4
Mantissa (8-bit, two's complement): 01011000
Step 5
Exponent +3 (4-bit two's complement): 0011
Answer
Mantissa: 01011000 Exponent: 0011

Negative Decimal → Floating Point

Example: Convert −23.25 (8-bit mantissa, 4-bit exponent)

Critical rule: Convert +23.25 to binary FIRST, normalise, THEN negate the whole mantissa using two's complement. Never negate the integer and fractional parts separately.
Step 1
+23.25 in binary: 23=10111, 0.25=.01 → 10111.01
Step 2
Normalise positive: 0.1011101 × 2⁵
Step 3
Positive 8-bit mantissa: 01011101
Step 4
Flip: 10100010 + 1 = 10100011
Check
Starts 10... ✓ (negative normalised)
Step 5
Exponent +5 (4-bit two's complement): 0101
Answer
Mantissa: 10100011 Exponent: 0101

Floating Point Binary → Decimal (Positive)

Example: 01101000 | 0011

Step 1
Mantissa 01101000 — MSB=0, positive
Step 2
Read as fraction: 0.1101000 = ½+¼+1/16 = 0.8125
Step 3
Exponent 0011 = +3
Step 4
Value = 0.8125 × 2³ = 0.8125 × 8 = 6.5
Answer
6.5

Floating Point Binary → Decimal (Negative)

Example: 10110000 | 0011

Step 1
Mantissa 10110000 — MSB=1, negative
Step 2
Flip: 01001111 +1 = 01010000
Step 3
Read as fraction: 0.1010000 = ½+⅛ = 0.625
Step 4
Exponent 0011 = +3
Step 5
Value = −0.625 × 2³ = −0.625 × 8 = −5.0
Answer
−5.0

🔑 Key Exam Tips – Floating Point
  • Always normalise BEFORE writing the mantissa — one common error is writing non-normalised bit patterns
  • Both mantissa AND exponent use two's complement independently
  • Negative exponent: the number is very small (e.g. exponent −3 means × 2⁻³ = × ⅛)
  • For negative decimals: convert positive first, normalise, THEN two's-complement the whole mantissa
  • Exam mark scheme: usually 1 mark for correct binary conversion, 1 for normalisation, 1 for both parts correct

Rounding, Absolute & Relative Errors

Why Errors Occur

Some decimal values have infinite binary representations and must be truncated. The stored value differs from the true value — this is a rounding error.

Absolute Error

Absolute Error = |Stored Value − True Value|

Example: True value = 6.625, stored = 6.5  →  |6.5 − 6.625| = 0.125

Relative Error

Relative Error = Absolute Error ÷ True Value    (× 100 for %)

Example: 0.125 ÷ 6.625 = 0.01887 = 1.887%

Why relative error matters more:
An error of 0.5 in 1,000,000 → relative error 0.00005% (negligible).
An error of 0.5 in 1.0 → relative error 50% (catastrophic).
Relative error lets you compare significance across very different magnitudes.

Summary of Error Types

Error TypeFormulaWhen to use
Absolute|stored − true|When you need the raw size of the error
Relativeabsolute ÷ trueWhen comparing errors at different magnitudes
Percentagerelative × 100Expressing relative error as a percentage
🔑 Common Exam Mistakes
  • Confusing absolute and relative error — remember: relative is a ratio
  • Not expressing relative error as a percentage when the question asks for %
  • Forgetting that rounding errors exist for BOTH fixed point AND floating point
  • Confusing overflow (too large for register) with "stack overflow" (completely different concept)
