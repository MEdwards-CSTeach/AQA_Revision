🖥️ 4.7.1 — Internal Hardware Components

4.7.1

The internal hardware components work together via buses to allow the CPU to communicate with memory and peripheral devices.

CPU

The processor — contains the ALU, Control Unit, clock and registers. Executes all instructions.

Main Memory (RAM)

Volatile temporary storage. Holds the currently running program and its data. Faster than secondary storage.

I/O Controllers

Connect peripheral devices to the system bus. Allow CPU to communicate with devices running at different speeds.

Addressable Memory

  • Each memory location has a unique binary address.
  • Width of the address bus determines how many locations can be addressed.
  • 32-bit bus: 232 = ~4.3 billion locations (4 GiB max).
  • 64-bit bus: 264 = enormous capacity (16 EiB theoretical max).
  • Each address typically refers to one byte in modern systems.

🚌 The System Bus

4.7.1

A bus is a shared set of wires connecting components to transfer data, addresses, or control signals.

Address Bus

  • Carries memory addresses from CPU to memory/devices
  • Direction: One-way →
  • Width determines how much memory can be addressed

Data Bus

  • Carries actual data or instructions between components
  • Direction: Two-way ↔
  • Width determines how much data is transferred at once

Control Bus

  • Carries control signals: read/write, clock, interrupt
  • Direction: Two-way ↔
  • Coordinates and manages all component activities
💡 Memory Jogger: "A Dog Carries" — Address carries location, Data carries content, Control carries signals.

⚖️ Von Neumann vs Harvard Architecture

4.7.1

Von Neumann

  • Instructions & data share the same memory
  • Single shared bus for code & data
  • Can cause the Von Neumann bottleneck
  • More flexible — memory is reusable
  • Used in: General-purpose computers, PCs

Harvard

  • Separate memory for instructions and data
  • Separate buses for code and data
  • Can fetch instruction AND data simultaneously
  • More complex hardware, less flexible
  • Used in: Embedded systems, DSP, microcontrollers
⚠️ Misconception Harvard is NOT simply "better" — each suits different contexts. Von Neumann still dominates general computing today.

💾 4.7.2 — The Stored Program Concept

4.7.2
"Machine code instructions stored in main memory are fetched and executed serially by a processor that performs arithmetic and logical operations."

Machine code

Programs must be in binary before the CPU can run them.

Stored in main memory

The program sits in RAM while it runs — not on disk, not hardwired.

Fetched and executed serially

Instructions are retrieved and run one at a time, in sequence.

Arithmetic & logical operations

The CPU can add, subtract, compare, AND, OR, NOT, etc.

💡 Historical note Before this concept, computers were reprogrammed by physically rewiring them (e.g. ENIAC). Von Neumann's insight: store the program as data in memory.

⚙️ 4.7.3 — CPU Components

4.7.3

ALU — Arithmetic Logic Unit

Performs all calculations:

  • Arithmetic: + − × ÷
  • Logical: AND, OR, NOT, XOR
  • Comparisons: greater than, equal to

💡 The Calculator

Control Unit (CU)

Controls and coordinates everything:

  • Decodes instructions
  • Sends control signals to all components
  • Manages the FDE cycle
  • Saves processor state during interrupts

💡 The Conductor

Clock

Generates regular electrical pulses:

  • Synchronises all CPU operations
  • Speed measured in Hz (GHz = billions/sec)
  • Each "tick" = one clock cycle
  • Faster clock = more cycles per second

💡 The Heartbeat

⚠️ Key distinction The Control Unit does NOT calculate — that is the ALU's job. The CU controls and coordinates; the ALU computes.

📝 CPU Registers

4.7.3

Registers are fast temporary storage inside the CPU.

Dedicated Registers

RegisterFull NamePurposeMnemonic
PCProgram CounterHolds the address of the NEXT instruction to be fetched📖 Bookmark
MARMemory Address RegisterHolds the address of the memory location being accessed✉️ Envelope address
MBRMemory Buffer RegisterHolds data just read from or about to be written to memory📬 In transit
CIRCurrent Instruction RegisterHolds the instruction currently being decoded and executed📋 Instruction on desk
SRStatus RegisterStores flags about result of last instruction: zero, carry, overflow, negative🚦 Traffic light

General-Purpose Registers

  • Fast temporary storage inside the CPU — programmer decides what to store in each.
  • Store values that instructions use as operands (inputs to the ALU).
  • Store the results of executing instructions (outputs from the ALU).
  • Store intermediate results during calculations.
  • A processor has a fixed number (e.g. R0, R1, R2…).
⚠️ Mark-scheme alert Do NOT say registers "store values inside the processor" — AQA mark schemes award NO marks for this. Say: "store values accessed quickly" or "store operands/results".
⚠️ Don't mix these up PC = address of NEXT instruction. CIR = current instruction. MAR = address. MBR = data.

🔄 The Fetch-Decode-Execute Cycle

4.7.3

1 — FETCH

  1. PC → MAR
  2. MAR → Address Bus → Memory
  3. PC increments (PC = PC + 1)
  4. Data Bus → MBR
  5. MBR → CIR

2 — DECODE

  1. CU reads CIR
  2. Splits into opcode + operand
  3. Opcode = WHAT to do
  4. Operand = data/address
  5. CU sends control signals

3 — EXECUTE

  1. ALU performs operation
  2. Memory read (if needed)
  3. Memory write (if needed)
  4. Result stored in register
  5. Status register updated

Fetch Stage — Detailed

  1. 1
    PC → MARContents of Program Counter copied to Memory Address Register. PC tells CPU where next instruction is stored.
  2. 2
    MAR → Address Bus → Main MemoryAddress bus carries this address — it is read-only (one-way).
  3. 3
    PC = PC + 1Program Counter incremented to point to the next instruction. Can happen simultaneously with steps 2 & 4.
  4. 4
    Memory → MBRInstruction at addressed memory location fetched into MBR via data bus.
  5. 5
    MBR → CIRContents of MBR copied into Current Instruction Register. Now ready to decode.

Execute Stage — Special Cases

  • If arithmetic/logical: ALU performs the operation; result stored in register.
  • If memory READ: address → MAR; data fetched via data bus → MBR; MBR → register.
  • If memory WRITE: data → MBR; address → MAR; write control signal sent.
  • If BRANCH (and condition is met): PC is updated to the branch address (not sequential).
💡 Analogy: Like reading a recipe — FETCH (find the step), DECODE (understand it), EXECUTE (do it!). Then repeat with the next step.

❌ Common Misconceptions — Don't Lose Marks!

Exam Technique
❌ Wrong: "The PC holds the current instruction" ✅ The PC holds the ADDRESS of the NEXT instruction. The CIR holds the current instruction.
❌ Wrong: "MAR and MBR do the same thing" ✅ MAR = address (where in memory). MBR = actual data (what is in that location).
❌ Wrong: "The Control Unit does the calculations" ✅ The ALU performs all calculations. The CU controls and coordinates — it does NOT compute.
❌ Wrong: "Registers store values inside the processor" ✅ AQA mark scheme: NO marks for this. Say: "store values accessed quickly" or "store operands/results".
❌ Wrong: "Harvard is always better than Von Neumann" ✅ Each suits different purposes. Von Neumann still dominates general computing.
