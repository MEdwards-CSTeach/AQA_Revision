4.5

Fundamentals of Data Representation

4.5.1 & 4.5.2 — Number Sets & Number Bases
Natural numbers (ℕ)
The set of positive integers including zero used for counting: {0, 1, 2, 3, …}
Integer (ℤ)
The set of all whole numbers, both positive and negative, including zero: {…, -2, -1, 0, 1, 2, …}
Rational number (ℚ)
Any number that can be expressed exactly as a fraction p/q where p and q are integers and q ≠ 0.
Irrational number
A number that cannot be expressed as an exact fraction; its decimal expansion is infinite and non-repeating (e.g. π, √2).
Real number (ℝ)
The complete set of all numbers on the number line: naturals, integers, rationals and irrationals.
Ordinal number
A number used to describe position/order within a set (1st, 2nd, 3rd …).
Binary (base 2)
A number system using only the digits 0 and 1. Each digit (bit) represents a power of 2.
Hexadecimal (base 16)
A number system using 16 symbols (0–9, A–F). One hex digit represents exactly four binary bits, making it a compact shorthand for binary.
Decimal (base 10)
The everyday number system using digits 0–9, where each position represents a power of 10.
4.5.3 — Units of Information
Bit
The fundamental unit of information: a single binary digit, either 0 or 1.
Byte
A group of 8 bits. The basic unit used to measure memory and storage capacity.
Kibibyte (KiB)
2¹⁰ bytes = 1,024 bytes. A binary prefix unit. Note: 1 kilobyte (kB) = 1,000 bytes (decimal prefix).
Mebibyte (MiB)
2²⁰ bytes = 1,048,576 bytes.
Gibibyte (GiB)
2³⁰ bytes ≈ 1.07 × 10⁹ bytes.
Tebibyte (TiB)
2⁴⁰ bytes ≈ 1.10 × 10¹² bytes.
4.5.4 — Binary Number System
Unsigned binary
A binary representation of non-negative integers only. With n bits the range is 0 to 2ⁿ – 1.
Signed binary (two's complement)
A binary representation that includes negative integers. The MSB has a negative place value. To negate: invert all bits then add 1. With n bits the range is –2ⁿ⁻¹ to 2ⁿ⁻¹ – 1.
Fixed-point binary
A binary representation for non-integer values where the binary point is always in the same position. Offers constant precision but limited range.
Floating-point binary
A binary representation using a mantissa (significant digits) and an exponent (scale). Allows a wider range of values at the cost of varying precision.
Mantissa
The significant-digit part of a floating-point number. Stored in two's complement; the binary point is assumed to follow the MSB.
Exponent
The power-of-two scale factor in floating-point representation. Stored in two's complement. Increasing the exponent shifts the binary point right (larger value).
Normalisation (floating point)
Adjusting the mantissa and exponent so the mantissa begins 0.1… (positive) or 1.0… (negative) in two's complement. Maximises precision for a given number of bits.
Rounding error
The inaccuracy introduced when a value cannot be represented exactly in a fixed number of bits. Some values (e.g. 0.1₁₀) have an infinitely repeating binary expansion.
Absolute error
The magnitude of the difference between the true value and the stored approximation: │true – stored│.
Relative error
The absolute error expressed as a proportion of the true value: │true – stored│ / │true│. Useful for comparing accuracy across different-sized numbers.
Overflow
Occurs when the result of a calculation is too large (positive or negative) to be stored in the available bits. The exponent cannot be increased further.
Underflow
Occurs when the result is too small (close to zero) to be represented; the non-zero mantissa would require an exponent smaller than the minimum.
4.5.5 — Information Coding Systems
ASCII
American Standard Code for Information Interchange. A 7-bit character encoding representing 128 characters. Extended ASCII uses 8 bits (256 characters).
Unicode
An international character encoding standard (up to 32 bits per character) that can represent over 1 million characters from all world scripts. Overcomes ASCII's limitation to English/Latin.
Character code
The numeric binary value assigned to a character in an encoding scheme (e.g. ASCII code for 'A' is 65₁₀ = 0100 0001₂).
Parity bit
An extra bit added to a group of bits to make the total number of 1s either always even (even parity) or always odd (odd parity). Used to detect single-bit transmission errors.
Majority voting
Each bit is transmitted multiple times (e.g. three). The receiver uses the value that appears in the majority of copies. Can detect and correct single-bit errors.
Checksum
A value computed from a block of data and transmitted with it. The receiver recomputes; a mismatch indicates corruption. Used in TCP/IP and file transfers.
Check digit
A digit appended to a number (e.g. ISBN, barcode) calculated from the other digits. Detects common data-entry errors such as transpositions.
Cipher
An algorithm used to perform encryption or decryption. Operates on plaintext to produce ciphertext.
Plaintext
The original, unencrypted data before any encryption algorithm is applied.
Ciphertext
The scrambled, unreadable data produced by applying an encryption algorithm to plaintext.
Caesar cipher
A simple substitution cipher in which each letter is shifted a fixed number of places in the alphabet. Easily cracked by frequency analysis or brute force (only 25 possible keys).
Vernam cipher (one-time pad)
A theoretically unbreakable cipher where the key is a random bit string at least as long as the message, used only once. The ciphertext carries no statistical information about the plaintext.
Computational security
Security based on the practical impossibility of cracking a cipher within a useful time frame, rather than mathematical proof of impossibility (as with Vernam).
4.5.6 — Representing Images, Sound & Other Data
Bit pattern
A sequence of bits. The same sequence can represent different things (number, character, colour, instruction) depending on context/data type.
Analogue data
Data that varies continuously and can take any value within a range (e.g. sound waves, temperature). Has infinite precision.
Digital data
Data represented as discrete values, typically binary 0s and 1s. Sampled and quantised from analogue sources.
ADC
Analogue-to-Digital Converter. Hardware that samples an analogue signal at regular intervals and converts each sample to a binary value. Used with microphones and sensors.
DAC
Digital-to-Analogue Converter. Hardware that converts a binary digital signal back to a continuous analogue signal. Most commonly used to convert digital audio to sound.
Bitmap (raster) graphic
An image stored as a 2D grid of pixels, each with a colour value encoded in binary. File size = width × height × colour depth (bits).
Pixel
The smallest addressable element of a raster image. Each pixel stores a colour value.
Resolution
The number of pixels in an image (width × height), or dots per inch (dpi). Higher resolution → more detail → larger file size.
Colour depth
The number of bits used to represent the colour of a single pixel. 8-bit gives 256 colours; 24-bit (true colour) gives over 16 million colours.
Vector graphic
An image stored as mathematical descriptions of geometric shapes. Scales without loss of quality; file size depends on complexity, not image size.
Sample rate (audio)
The number of times per second an analogue audio signal is measured. Measured in Hz. Higher sample rate → more accurate reproduction → larger file.
Sample resolution / bit depth
The number of bits used to store each audio sample. More bits → finer volume gradations → higher quality and larger file size.
MIDI
Musical Instrument Digital Interface. Stores music as instructions (note, duration, instrument) rather than recorded sound. Very compact; quality depends on playback hardware.
Lossless compression
Compression that allows the original data to be perfectly reconstructed. No information is lost (e.g. RLE, ZIP).
Lossy compression
Compression that permanently removes some data, reducing quality but achieving smaller file sizes (e.g. JPEG, MP3). Cannot be reversed.
Run-length encoding (RLE)
A lossless compression method that replaces consecutive repeated values with a count and the value (e.g. AAABBBB → 3A 4B).
Dictionary-based compression
Lossless compression that builds a dictionary of repeated sequences and replaces them with shorter codes (e.g. LZ77/LZW used in ZIP and GIF).

4.6

Fundamentals of Computer Systems

4.6.1 — Hardware & Software
Hardware
The physical, tangible components of a computer system (e.g. CPU, RAM, hard drive, keyboard).
Software
Programs (sets of instructions) that run on hardware and make the computer useful. Can be changed without altering physical components.
System software
Software that provides a platform for running application software and manages hardware resources (e.g. operating systems, device drivers, utilities).
Application software
Software designed to help users perform specific tasks (e.g. word processors, games, web browsers).
Operating system (OS)
System software that acts as an interface between the user/applications and the hardware. Manages resources (CPU time, memory, I/O) among competing processes and hides hardware complexity.
Utility program
System software performing a specific maintenance or housekeeping task (e.g. disk defragmenter, antivirus, file compression tool).
Library
Pre-compiled code that can be called by other programs to perform common tasks without rewriting the code (e.g. maths libraries, graphics libraries).
4.6.2–4.6.3 — Programming Languages & Translators
Machine code
The lowest-level programming language consisting of binary instructions (0s and 1s) that can be executed directly by the CPU. Processor-specific; not portable.
Assembly language
A low-level language using human-readable mnemonics (e.g. MOV, ADD) that correspond 1-to-1 with machine code instructions. Converted to machine code by an assembler.
Imperative high-level language
A programming language (e.g. Python, Java) that uses English-like statements and abstracts away hardware details. Translated to machine code by a compiler or interpreter.
Assembler
A translator that converts assembly language source code directly into machine code, one instruction at a time (one-to-one translation).
Compiler
A translator that converts the entire high-level source code into machine code in one go before execution. Produces a standalone executable; errors reported after full analysis.
Interpreter
A translator that executes high-level source code line by line at runtime, translating and running each statement before moving to the next. No standalone executable is produced.
Bytecode
An intermediate form of compiled code (e.g. Java .class files) that is not native machine code but is executed by a virtual machine (JVM). Enables platform independence.
Source code
The human-readable version of a program written by the programmer, before translation.
Object (executable) code
The machine-code output produced by a compiler; ready to be loaded and run by the CPU.
4.6.4–4.6.5 — Logic Gates & Boolean Algebra
Logic gate
An electronic component that implements a Boolean function, taking one or more binary inputs and producing a single binary output.
NOT gate
Inverts the input: output = 1 when input = 0, and vice versa. Boolean expression: Ā (or ¬A).
AND gate
Output = 1 only when ALL inputs are 1. Boolean: A · B (or A AND B).
OR gate
Output = 1 when AT LEAST ONE input is 1. Boolean: A + B (or A OR B).
XOR gate
Exclusive OR: output = 1 when inputs are DIFFERENT. Boolean: A ⊕ B.
NAND gate
NOT AND: output = 0 only when all inputs are 1. Boolean: ¬(A · B). Functionally complete — all other gates can be built from NAND gates.
NOR gate
NOT OR: output = 1 only when ALL inputs are 0. Boolean: ¬(A + B). Also functionally complete.
Truth table
A table showing all possible input combinations and the corresponding output for a logic circuit or Boolean expression.
Half-adder
A circuit that adds two single bits, producing a Sum output (XOR) and a Carry output (AND). Cannot handle a carry-in.
Full-adder
A circuit that adds two bits plus a carry-in, producing a Sum and Carry-out. Built from two half-adders. Used in multi-bit addition circuits.
D-type flip-flop
An edge-triggered memory element that stores one bit. The stored value updates to the input (D) on the rising (or falling) edge of a clock signal.
Boolean algebra
A mathematical system for analysing and simplifying logic expressions using identities and laws (commutative, associative, distributive, De Morgan's).
De Morgan's laws
Two rules: (1) ¬(A · B) = ¬A + ¬B and (2) ¬(A + B) = ¬A · ¬B. Used to convert between NAND and NOR expressions and to simplify circuits.

4.7

Fundamentals of Computer Organisation & Architecture

4.7.1 — Internal Hardware Components
Processor (CPU)
The Central Processing Unit: the component that executes program instructions. Contains the ALU, control unit, registers and cache.
Arithmetic Logic Unit (ALU)
The part of the CPU that performs arithmetic operations (add, subtract, multiply) and logical operations (AND, OR, NOT, compare).
Control Unit (CU)
The part of the CPU that directs and coordinates all operations. It fetches instructions, decodes them and issues control signals to execute them.
Clock
An oscillator that generates a regular pulse (measured in Hz/GHz). Each pulse synchronises CPU operations; a higher clock speed allows more instructions per second.
General-purpose register
A small, fast storage location inside the CPU used to hold data or addresses being actively processed (e.g. ACC in AQA assembly).
Program Counter (PC)
A dedicated register that holds the memory address of the next instruction to be fetched.
Current Instruction Register (CIR)
Holds the instruction currently being decoded and executed.
Memory Address Register (MAR)
Holds the address in main memory that is about to be read from or written to.
Memory Buffer Register (MBR/MDR)
A temporary store for data being transferred between the CPU and main memory.
Status Register (SR)
Contains flag bits recording the outcome of the last operation (e.g. zero flag, carry flag, overflow flag).
Main memory (RAM)
Volatile, directly addressable storage holding programs and data currently in use. Slower than registers/cache but much larger.
Address bus
A unidirectional bus carrying memory addresses from the CPU to memory/I/O. Width determines the maximum addressable memory (2ⁿ locations for n-bit bus).
Data bus
A bidirectional bus carrying data between the CPU, memory and I/O devices. Width determines how many bits are transferred at once.
Control bus
A bidirectional bus carrying control signals (e.g. memory read/write, interrupt request, clock). Coordinates all components.
I/O controller
An interface between the CPU and peripheral devices. Translates signals, buffers data, handles interrupts and carries out I/O-related processing to reduce CPU workload.
Von Neumann architecture
A computer design in which a single, shared memory stores both instructions and data, connected to the CPU by a single bus. Simple but subject to the "von Neumann bottleneck".
Harvard architecture
A design with physically separate memory and buses for instructions and data. Allows simultaneous fetch of instruction and data; used in embedded/DSP systems.
Addressable memory
Each location in memory has a unique address. The CPU uses the address bus to specify which location to read from or write to.
4.7.2–4.7.3 — Stored Program Concept & Fetch-Execute Cycle
Stored program concept
The principle that machine-code instructions stored in main memory are fetched and executed serially by the processor. Programs can be changed by modifying memory contents, without altering hardware.
Fetch-Decode-Execute cycle
The repeating sequence by which the CPU processes instructions: (1) Fetch – copy instruction at address in PC into CIR, increment PC; (2) Decode – interpret the instruction; (3) Execute – carry out the instruction.
Interrupt
A signal to the CPU from hardware or software requesting immediate attention (e.g. I/O complete, hardware fault). The CPU saves its current state, handles the interrupt via an ISR, then resumes.
Interrupt Service Routine (ISR)
A small program that handles a specific type of interrupt. Executed when that interrupt is triggered.
Pipelining
A technique where multiple stages of the fetch-decode-execute cycle are overlapped for different instructions simultaneously, increasing throughput.
Cache memory
Small, very fast memory located close to the CPU that stores copies of frequently accessed data/instructions to reduce the average time to access main memory.
4.7.4 — Instruction Set & Assembly Language
Instruction set
The complete set of machine-code instructions that a particular CPU can understand and execute. Different processor families have different instruction sets (not portable).
Opcode
The part of a machine-code instruction that specifies WHAT operation to perform (e.g. load, add, store).
Operand
The part of a machine-code instruction that specifies WHERE the data is (memory address, register, or immediate value) on which the operation acts.
Addressing modes
The different ways an operand can specify where data is located: immediate (literal value), direct (memory address), indirect (address of address), indexed (base address + offset).
Immediate addressing
The operand IS the actual data value to be used (e.g. LDR R0, #5 loads the value 5).
Direct addressing
The operand is a memory address; the CPU reads the data stored at that address.
Indirect addressing
The operand is an address that contains another address, which holds the actual data.
RISC
Reduced Instruction Set Computer. A CPU design philosophy with a small set of simple, fixed-length instructions that each execute in one clock cycle. Favours registers over complex memory operations.
CISC
Complex Instruction Set Computer. A CPU design with a large set of complex, variable-length instructions; a single instruction may perform several operations. Examples: x86 architecture.
Co-processor
A secondary processor that supplements the main CPU by handling specific tasks (e.g. graphics GPU, floating-point unit FPU).
GPU (Graphics Processing Unit)
A massively parallel co-processor originally designed for rendering graphics, now also used for scientific computing and machine learning due to its thousands of smaller cores.
Multicore processor
A single chip containing two or more independent processor cores, enabling true parallel execution of multiple instruction streams.
4.7.5 — External Hardware Devices
Secondary storage
Non-volatile storage (data persists when power is off) used for long-term data storage: HDD, SSD, optical disc, flash/USB.
HDD (Hard Disk Drive)
Magnetic storage: platters coated with magnetisable material spin at high speed; read/write heads move radially. Data stored in tracks divided into sectors. Relatively slow random access.
SSD (Solid-State Drive)
Flash-based non-volatile storage with no moving parts. Faster than HDD, more durable, but historically more expensive per GB.
Optical storage
Uses a laser to read/write data as pits and lands on a reflective disc (CD, DVD, Blu-ray). Suitable for distribution and archiving.
Flash/USB storage
NAND flash memory packaged as USB drives or memory cards. Portable, non-volatile, no moving parts; limited write cycles.
RFID
Radio-Frequency Identification. Uses radio waves to read/write data on tags attached to objects. No line-of-sight needed; used in contactless payments, stock control, security.

4.8

Consequences of Uses of Computing

Computer Misuse Act 1990
UK legislation making it illegal to: (1) access computer material without authorisation, (2) access with intent to commit a further crime, (3) unauthorised modification of computer material (e.g. planting malware).
Data Protection Act 2018 / UK GDPR
Law governing how personal data must be collected, stored and processed. Key principles: data must be fairly and lawfully processed, accurate, kept only as long as necessary, and kept securely.
Copyright, Designs and Patents Act 1988
Protects intellectual property: code, images, music and text are automatically copyright. Copying, distributing or modifying without permission is illegal unless under a licence.
RIPA 2000
Regulation of Investigatory Powers Act. UK law governing surveillance and interception of communications. Allows law enforcement to intercept communications under warrant; raises privacy concerns.
Creative Commons licence
A licensing system allowing creators to grant others rights to share, adapt or use their work, sometimes with conditions (e.g. attribution, non-commercial, share-alike).
Open-source software
Software whose source code is publicly available for anyone to view, use, modify and distribute, usually under a licence such as GPL. Examples: Linux, Firefox.
Proprietary software
Software owned by an individual or company; source code is not freely available and use is governed by a licence agreement.
Digital divide
The gap between those who have access to digital technology and the internet and those who do not, often correlating with wealth, geography or age.
Malware
Software designed to disrupt, damage or gain unauthorised access to a system. Includes viruses, worms, trojans, ransomware, spyware and adware.
Virus
Malware that attaches itself to a legitimate program and replicates by infecting other programs when executed. Requires user action to spread.
Worm
Self-replicating malware that spreads automatically across networks without needing to attach to a host program. Exploits network/OS vulnerabilities.
Trojan horse
Malware disguised as legitimate software. Does not self-replicate, but opens a backdoor or causes damage once executed by the user.
Ransomware
Malware that encrypts a victim's files and demands payment (ransom) in exchange for the decryption key.
Phishing
A social engineering attack using fraudulent emails, websites or messages to trick users into revealing passwords, financial details or installing malware.
4.9

Fundamentals of Communication & Networking

4.9.1 — Communication
Serial transmission
Data bits are sent one after another along a single wire. Slower but cheaper and suitable for long distances; used by USB.
Parallel transmission
Multiple bits sent simultaneously along multiple wires. Faster over short distances but susceptible to interference/skew; used internally (e.g. data bus).
Synchronous transmission
Data is sent at a fixed rate controlled by a shared clock signal between sender and receiver. Efficient for continuous data streams.
Asynchronous transmission
Data is sent without a shared clock; start and stop bits frame each byte to synchronise sender and receiver. Used for irregular data (e.g. keyboard input).
Baud rate
The number of signal changes (symbols) per second transmitted over a channel. Measured in baud.
Bit rate
The number of bits per second transmitted. Bit rate ≥ baud rate when more than one bit is encoded per signal change.
Bandwidth
The maximum data transfer rate of a channel, measured in bits per second (bps). Bit rate is directly proportional to bandwidth.
Latency
The delay between sending and receiving data; the time for a signal to travel from source to destination.
Protocol
A set of agreed rules governing communication between devices (e.g. format, timing, error checking, synchronisation).
4.9.2 — Networking
LAN
Local Area Network. A network covering a small geographical area (e.g. single building) owned and managed by one organisation.
WAN
Wide Area Network. A network spanning a large geographical area, typically made up of interconnected LANs (the Internet is the largest WAN).
Physical (star) topology
A network layout where each device connects directly to a central switch/hub via dedicated cable. Failure of one link does not affect others.
Logical (bus) topology
A network behaviour in which data is broadcast to all devices on a shared medium; only the intended recipient processes each frame.
Peer-to-peer (P2P) network
Each device has equal status and can both request and provide resources. Simple, no dedicated server needed; less scalable than client-server.
Client-server network
One or more servers provide services (files, email, web) to multiple client machines. Centralised management; scales well.
Switch
A network device that connects multiple devices in a LAN and uses MAC address tables to forward frames only to the correct port.
Router
A network device that forwards packets between networks (e.g. LAN to Internet), using IP addresses and routing tables.
MAC address
Media Access Control address: a unique 48-bit hardware identifier assigned to every NIC at manufacture. Used at the link layer.
CSMA/CA
Carrier Sense Multiple Access with Collision Avoidance. A wireless protocol where a device listens before transmitting and waits a random time if the channel is busy, to reduce collisions.
RTS/CTS
Request To Send / Clear To Send. An optional addition to CSMA/CA where a device sends an RTS frame before transmitting; only proceeds when it receives CTS from the access point.
4.9.3 — The Internet
Packet switching
A method of data transmission where data is broken into packets, each sent independently across the network and reassembled at the destination. Efficient and fault-tolerant.
Packet
A unit of data transmitted over a network, containing: a header (source IP, destination IP, sequence number, checksum) and the payload (data).
DNS (Domain Name System)
A hierarchical distributed database that translates domain names into IP addresses. Acts like an internet phone book.
Firewall
Security hardware/software that monitors and controls incoming and outgoing network traffic according to rules. Methods: packet filtering, proxy server, stateful inspection.
Proxy server
A server acting as an intermediary between client and internet. Caches web pages, filters content and hides clients' IP addresses.
Symmetric encryption
Encryption where the same key is used to encrypt and decrypt data. Fast, but the key must be securely shared beforehand (e.g. AES).
Asymmetric (public-key) encryption
Uses a mathematically linked key pair: a public key (shared openly, encrypts) and a private key (kept secret, decrypts). Solves the key-exchange problem (e.g. RSA).
Digital certificate
An electronic document issued by a Certificate Authority (CA) that links a public key to the identity of its owner. Used in HTTPS to verify website authenticity.
Digital signature
A cryptographic mechanism where the sender encrypts a message hash with their private key. The receiver decrypts with the sender's public key to verify authenticity and integrity.
4.9.4 — TCP/IP
TCP/IP model
A four-layer model describing how data is transmitted over the internet: Application, Transport, Network/Internet, Link (physical).
Application layer
The topmost TCP/IP layer. Provides network services directly to user applications. Protocols: HTTP, HTTPS, FTP, SMTP, POP3, SSH, DNS.
Transport layer
Manages end-to-end communication between hosts. Breaks data into segments, adds port numbers, handles error checking and flow control. Protocols: TCP, UDP.
Network / Internet layer
Routes packets across networks using IP addresses. Adds source and destination IP addresses. Protocol: IP.
Link layer
Handles physical transmission of data on the local network. Adds MAC addresses. Involves device drivers and network hardware (NIC).
TCP
Transmission Control Protocol. A connection-oriented transport protocol that guarantees reliable, ordered delivery using acknowledgements, sequencing and retransmission of lost packets.
UDP
User Datagram Protocol. A connectionless, unreliable transport protocol with low overhead. No acknowledgements or retransmission; used where speed is critical (e.g. video streaming, DNS).
Port number
A 16-bit number identifying a specific process/service on a host. Well-known ports: HTTP=80, HTTPS=443, FTP=21, SMTP=25, POP3=110, SSH=22.
IP address
A unique numeric label assigned to every device on a network. IPv4: 32-bit (e.g. 192.168.1.1). IPv6: 128-bit. Split into network and host identifier parts.
Subnet mask
A 32-bit value used with an IP address to distinguish the network identifier bits from the host identifier bits (e.g. 255.255.255.0 = /24).
NAT
Network Address Translation. A router function mapping multiple private IP addresses to a single public IP address. Allows many devices to share one public IP; adds a security layer.
DHCP
Dynamic Host Configuration Protocol. Automatically assigns IP addresses, subnet masks, gateways and DNS server addresses to devices joining a network.

4.10

Fundamentals of Databases

4.10.1–4.10.3 — Data Modelling & Normalisation
Entity
A distinct object or concept about which data is stored (e.g. Student, Course, Order). Corresponds to a table in a relational database.
Attribute
A property or characteristic of an entity that is stored as a column in a database table (e.g. StudentID, Name, DateOfBirth).
Entity Relationship (ER) diagram
A diagram showing entities, their attributes and the relationships between them, using standard notation (rectangles for entities, lines for relationships).
Primary key
An attribute (or set of attributes) that uniquely identifies each record/row in a table. Cannot be NULL; must be unique.
Composite primary key
A primary key made up of two or more attributes that together uniquely identify a record, where no single attribute alone is sufficient.
Foreign key
An attribute in one table that references the primary key of another table. Enforces referential integrity by linking related tables.
Referential integrity
The rule that every foreign key value must match an existing primary key value in the referenced table (or be NULL). Prevents orphaned records.
Normalisation
The process of structuring a relational database to reduce data redundancy and improve data integrity, using a series of normal forms (1NF, 2NF, 3NF).
First Normal Form (1NF)
A table is in 1NF if: all attributes contain atomic (indivisible) values, there are no repeating groups, and every row is uniquely identifiable by a primary key.
Second Normal Form (2NF)
A table is in 2NF if it is in 1NF AND every non-key attribute is fully functionally dependent on the WHOLE primary key (eliminates partial dependencies).
Third Normal Form (3NF)
A table is in 3NF if it is in 2NF AND no non-key attribute is transitively dependent on the primary key. Every non-key attribute depends only on the key.
Functional dependency
Attribute B is functionally dependent on attribute A if knowing A determines the value of B (written A → B).
Partial dependency
A non-key attribute depends on only PART of a composite primary key. Violates 2NF.
Transitive dependency
A non-key attribute depends on another non-key attribute, rather than directly on the primary key. Violates 3NF.
4.10.4 — SQL
SELECT … FROM … WHERE
The fundamental SQL query structure to retrieve data: SELECT (choose columns), FROM (choose table), WHERE (filter rows by condition).
JOIN
Combines rows from two or more tables based on a related column. INNER JOIN returns only matching rows; LEFT JOIN includes all rows from the left table.
INSERT INTO
SQL command to add new rows to a table: INSERT INTO table (col1, col2) VALUES (val1, val2);
UPDATE … SET
SQL command to modify existing data: UPDATE table SET column = value WHERE condition;
DELETE FROM
SQL command to remove rows: DELETE FROM table WHERE condition; (without WHERE deletes ALL rows).
ORDER BY
Clause sorting query results by one or more columns: ORDER BY column ASC|DESC.
GROUP BY / HAVING
GROUP BY groups rows with identical values in specified columns, used with aggregate functions. HAVING filters groups produced by GROUP BY (like WHERE but for aggregates).
Aggregate functions
Functions that compute a single result from multiple rows: COUNT(), SUM(), AVG(), MAX(), MIN().
4.10.5 — Client-Server Databases & Concurrency
Concurrent access
Multiple users reading or writing the same database records at the same time. Without control, this can corrupt data (lost updates).
Record lock
Prevents other transactions from reading or writing a record while it is being updated, until the transaction completes.
Serialisation
Ensuring transactions execute as if they ran one at a time, even when actually concurrent, to prevent inconsistencies.
Timestamp ordering
Each transaction is given a timestamp; access to records is controlled to ensure transactions appear to execute in timestamp order.

4.11

Big Data

Big Data
A term for datasets so large or complex that traditional database tools cannot store, manage or process them within an acceptable time.
Volume
The sheer size of Big Data – too large to fit on a single server. Requires distributed storage across many machines.
Velocity
The speed at which data is generated and must be processed (e.g. streaming data from sensors, social media – milliseconds to seconds response time).
Variety
The diverse formats of Big Data: structured (tables), unstructured (video, audio, text), semi-structured (JSON, XML). Lack of structure is the greatest challenge.
Veracity
The trustworthiness and quality of data – uncertainty, inconsistency and noise in large datasets.
Value
The usefulness or insight that can be extracted from the data to support decision-making.
Distributed processing
Spreading computation across multiple machines/servers to handle datasets too large for one machine.
Machine learning
Algorithms that learn patterns from large datasets without being explicitly programmed. Essential for extracting insight from unstructured Big Data.
Fact-based model
A data representation where each item of information is stored as an individual, atomic fact, rather than grouped in rows/columns. Suits unstructured Big Data.
Graph schema
A structure for Big Data using nodes (entities), edges (relationships between nodes) and properties (attributes of nodes/edges).
Immutable data
Data that cannot be changed after creation. New results are created rather than modifying existing data, preventing race conditions in distributed systems.
Statelessness
A function/system property where operations do not depend on stored state from previous calls. Each call is independent, simplifying distributed execution.
4.12

Fundamentals of Functional Programming

4.12.1 — Functional Programming Paradigm
Functional programming
A programming paradigm that treats computation as the evaluation of mathematical functions, avoiding changing state and mutable data.
Function (in FP)
A mapping from a domain (set of inputs) to a co-domain (set of possible outputs). Written as f: A → B where A is the domain and B is the co-domain.
Domain
The set of values that can be used as input to a function (the argument type).
Co-domain
The set from which output values of a function are drawn. Not all co-domain values need to be outputs (contrast with range/image).
First-class function
Functions that are treated as values: they can be assigned to variables, passed as arguments to other functions and returned as results.
Higher-order function
A function that takes one or more functions as arguments and/or returns a function as its result (e.g. map, filter, fold/reduce).
Function composition
Combining two functions f and g to create a new function h where h(x) = f(g(x)). Written as f ∘ g.
Partial function application
Supplying fewer arguments than a function expects, producing a new function that takes the remaining arguments.
Currying
Transforming a function that takes multiple arguments into a sequence of functions each taking a single argument (named after logician Haskell Curry).
Pure function
A function whose output depends only on its inputs and has no side effects. Always returns the same result for the same arguments.
Side effect
Any effect a function has beyond returning a value (e.g. modifying a global variable, writing to a file, printing to screen). Functional programming avoids side effects.
Referential transparency
A property where an expression can be replaced with its value without changing the program's behaviour. Requires pure functions.
4.12.2 — Writing Functional Programs
List
The fundamental data structure in functional programming. A list can be empty ([]) or consist of a head (first element) and a tail (the rest of the list).
Head
The first element of a non-empty list.
Tail
All elements of a list except the first. The tail of a single-element list is an empty list.
Recursion (in FP)
The main technique for repetition in functional programming (loops are avoided). A function calls itself on a smaller problem until a base case is reached.
Base case
The condition in a recursive function that terminates recursion (e.g. an empty list). Without a base case, recursion would be infinite.
Map
A higher-order function that applies a given function to every element of a list, returning a new list of results: map f [a, b, c] = [f(a), f(b), f(c)].
Filter
A higher-order function that returns a new list containing only those elements of a list for which a predicate function returns True.
Fold (reduce)
A higher-order function that combines all elements of a list into a single result by repeatedly applying a binary function with an accumulator (e.g. summing a list).
Predicate
A function that returns a Boolean value (True or False). Used by filter to test whether each element should be kept.
Lambda (anonymous) function
A nameless function defined inline, typically with a brief body. Written in Haskell as \x -> expression, in Python as lambda x: expression.
