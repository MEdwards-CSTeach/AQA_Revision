Section 1

Trees, Graphs & How They Differ

What is a Graph?

  • A data structure of vertices (nodes) connected by edges (arcs)
  • Can be directed (arrows) or undirected
  • Can be weighted (edges have costs/distances)
  • May contain cycles – paths that loop back to the start
  • Can be disconnected – not all nodes need to be reachable

Graph Types & Representations

  • Undirected: Facebook friends
  • Directed: Twitter follows
  • Weighted: Road map distances
  • Unweighted: Network topology

Represented as: Adjacency matrix · Adjacency list

What is a Tree?

  • A tree is a connected, undirected graph with NO cycles
  • N nodes → exactly N−1 edges
  • Any two nodes are connected by exactly one path
  • A tree does not have to have a root node
  • Rooted tree: one node designated as root; parent-child relationships exist
  • All nodes except the root have exactly one parent

Key Tree Terms

  • Root – topmost node
  • Leaf – node with no children
  • Height – longest root-to-leaf path
  • Depth – distance from root
  • Subtree – any node + descendants

Binary Search Trees

Binary Trees

  • Each node has at most two children (left & right)
  • Full binary tree: every node has 0 or 2 children
  • Complete binary tree: all levels full except possibly the last
  • Maximum nodes at depth d = 2d
  • Binary tree with height h has at most 2(h+1) − 1 nodes

Binary tree structure. Each node stores: data value · left pointer · right pointer

BST Ordering Rule

The Golden Rule (applies recursively): LEFT subtree values < node value < RIGHT subtree values
  • No duplicate values (typically)
  • Enables efficient O(log n) search, insert, delete
  • In-order traversal gives values in ascending sorted order

BST Example – Insert: 15, 9, 22, 5, 12, 19, 27

BST Operations

Search Algorithm

  1. Start at root
  2. If value = node → FOUND
  3. If value < node → go LEFT
  4. If value > node → go RIGHT
  5. If null pointer → NOT FOUND

Time complexity: O(log n) average · O(n) worst case (unbalanced)

Insert Algorithm

  1. Search for insertion point
  2. When null pointer reached, create new node there
  3. If value < parent → left child
  4. If value > parent → right child
  5. Duplicates: typically rejected or go right
⚡ BST vs Linear Search For 1 million items: BST ≈ 20 comparisons vs Linear ≈ 500,000 comparisons. Worst case: a BST built from already-sorted data degenerates to a linked list → O(n). Self-balancing trees (AVL, Red-Black) prevent this.

Section 3

Tree Traversal Algorithms

Pre-order (NLR)

Visit Node FIRST, then Left subtree, then Right subtree.

Root → Left → Right

Output (example tree): 17, 9, 5, 13, 22, 19, 27

Uses: Copying a tree · Prefix notation · Serialising a tree

In-order (LNR)

Visit Left subtree, then Node, then Right subtree.

Left → Root → Right

Output (example tree): 5, 9, 13, 17, 19, 22, 27 ← sorted!

Uses: BST sorted output · Infix notation from expression tree

Post-order (LRN)

Visit Left subtree, then Right subtree, then Node LAST.

Left → Right → Root

Output (example tree): 5, 13, 9, 19, 27, 22, 17

Uses: Deleting a tree · Postfix/RPN notation · Evaluating expressions

Traversal Worked Example – Insert: 17, 9, 22, 5, 13, 19, 27

Traversal Results

Pre-order (NLR):
17, 9, 5, 13, 22, 19, 27

In-order (LNR) – sorted! ✓
5, 9, 13, 17, 19, 22, 27

Post-order (LRN):
5, 13, 9, 19, 27, 22, 17

Memory Aid Pre = Node first ("preview the root before you go anywhere")
In = Node between left and right ("in the middle")
Post = Node last ("post = after everything else")

Traversal Uses – AQA Specification

  • Copying/cloning a tree

  • Producing prefix (Polish) notation from expression tree

  • Printing BST values in ascending sorted order

Section 4

Infix, Prefix & Postfix Notation

Infix Notation

Operator is BETWEEN operands

What humans use!

A + B (A + B) * C 3 + 4 * 2

Needs brackets to show precedence. Ambiguous without brackets or rules.

Prefix (Polish) Notation

Operator comes BEFORE operands

Invented by Jan Łukasiewicz (1924)

+ A B * + A B C + 3 * 4 2

No brackets needed! Unambiguous. Tree pre-order gives prefix.

Postfix (RPN) Notation

Operator comes AFTER operands

Reverse Polish Notation (RPN)

A B + A B + C * 3 4 2 * +

No brackets needed! Evaluated with a stack. Tree post-order gives postfix.

Converting Between Notations – Expression: (A + B) * (C − D)

Why Postfix for Computers? Stack-based evaluation: scan left to right, push operands, pop and apply operators when seen. No brackets or precedence rules needed at evaluation time. Used in: PostScript, Java bytecode, HP calculators, many interpreters.

Operator Rule

When you see an operator:

  1. Pop RIGHT operand
  2. Pop LEFT operand
  3. Apply operator
  4. Push result

Final value on stack = answer

Prefix evaluation: scan right to left (push operands, pop two operands when operator found, apply, push result)

RPN & Expression Trees

Expression Trees

  • A binary tree representing a mathematical expression
  • Internal nodes (non-leaf) hold operators (+, −, *, /)
  • Leaf nodes hold operands (numbers or variables)
  • Tree structure encodes operator precedence without brackets
  • The three traversals give the three notations directly
  • Used in compilers for Abstract Syntax Trees (AST)

Example: (3 + 4) * (5 − 2)

Pre-order: * + 3 4 - 5 2

In-order: 3 + 4 * 5 - 2

Post-order: 3 4 + 5 2 - *

Infix to RPN – Full Worked Example

Convert: (5 + 3) * 2 − 8 / 4

Expression Tree

Post-order Traversal → RPN

Visit left subtree of −:
  Post-order of *: 5 3 + 2 *

Visit right subtree of −:
  Post-order of /: 8 4 /

Visit root: −

RPN: 5 3 + 2 * 8 4 / −

Verify: (5+3)×2 − 8/4 = 16−2 = 14 ✓

Why Use RPN / Postfix?

  • No brackets needed – operator position uniquely determines evaluation order
  • No precedence rules at evaluation time; just use a stack
  • Stack-based CPUs – RPN maps naturally to push/pop instructions
  • Real-world uses: PostScript (laser printers), Forth language, HP calculators, Java bytecode, compiler intermediate representations

⭐ Key Exam Points – Summary

Graph vs Tree
Graph = vertices + edges; may have cycles. Tree = connected, undirected, NO cycles. N nodes → N−1 edges.
BST Property
Left < node < right (recursive). In-order traversal gives sorted output. O(log n) average search.
Traversal Orders
Pre-order (NLR): copy tree, prefix. In-order (LNR): sorted output, infix. Post-order (LRN): delete tree, RPN.
Notations
Infix: operator between (needs brackets). Prefix: operator first. Postfix/RPN: operator last, stack-evaluated.
RPN Advantages
No brackets, no precedence ambiguity, stack-based evaluation, used in compilers & PostScript.
Expression Trees
Pre-order = prefix, in-order = infix, post-order = postfix (RPN). Internal nodes = operators, leaves = operands.
