📊 Barcode Reader — Principles of Operation

4.7.4.1

Method 1 — Laser

  • Laser fires and mirror sweeps beam across barcode
  • Black bars absorb light; white bars reflect light
  • Photodiode measures reflected light intensity
  • ADC converts the analogue signal to binary

Method 2 — CCD (Camera)

  • CCD/CMOS grid captures an image of the barcode
  • Each pixel measures light intensity
  • Voltage output from each pixel
  • ADC converts voltage to binary

Suitability

  • ✅ Fast, accurate
  • ❌ Needs line-of-sight
  • ❌ Cannot read damaged/dirty barcodes

Typical Uses

  • Supermarket checkouts
  • Libraries
  • Warehouses
  • Hospitals
⚠️ Key exam point Barcodes store a product NUMBER only — NOT the name, price or description. The till looks these up in a database.
💡 Exam tip You only need to describe ONE method in exams (laser OR camera). Know both exist!

📡 RFID — Radio-Frequency Identification

4.7.4.1
  1. ①
    Reader transmitsEmits an electromagnetic field/signal
  2. ②
    Tag energisedField induces current in passive tag's antenna (NO battery needed)
  3. ③
    Tag sends dataTag transmits stored ID data via radio waves
  4. ④
    Reader decodesConverts radio signal back to binary data

RFID vs Barcode

  • ✅ No line-of-sight needed
  • ✅ Can read multiple tags simultaneously
  • ❌ Higher cost per tag than barcode

Active vs Passive Tags

  • Passive tag: NO battery — gets power from the reader's electromagnetic field
  • Active tag: Has its own battery, longer range
💡 Mnemonic: RFID = Radio, Field, ID — Reader sends out a radio field → tag gets power → tag sends its ID back.

📷 Digital Camera — Principles of Operation

4.7.4.1
  1. 1
    Light → LensLight from scene passes through the lens and is focused onto the sensor
  2. 2
    CCD/CMOS SensorGrid of millions of tiny pixel cells — each measures light intensity
  3. 3
    Voltage outputEach pixel produces a voltage proportional to light intensity received
  4. 4
    ADC convertsAnalogue voltages converted to binary values by an Analogue-to-Digital Converter
  5. 5
    Stored as bitmapPixel colour values stored as a bitmap in flash memory (non-volatile)
⚠️ Resolution Resolution = megapixels (millions of pixels). More pixels = more detail captured but larger file size. NOT the same as "good lens quality".
💡 Mnemonic — LIVABS: Light → Image (sensor) → Voltage → ADC → Binary → Store

🖨️ Laser Printer — Principles of Operation

4.7.4.1
  1. ①
    Build bitmapBitmap of page built in printer's memory from page description
  2. ②
    Charge drum (−)Negative charge applied uniformly to entire photosensitive drum
  3. ③
    Laser firesLaser (via rotating mirror) neutralises charge where image will appear — the laser hits the DRUM, not the paper
  4. ④
    Toner appliedNegatively charged toner (fine powder) sticks to discharged (laser-hit) areas
  5. ⑤
    Transfer to paperPositively charged transfer roller pulls toner from drum onto paper
  6. ⑥
    Fuser heatsFuser (heater) permanently melts/fuses toner powder onto paper
⚠️ Critical exam point The laser NEVER touches the paper. Toner is a fine POWDER — NOT ink. The laser only hits the drum, not the paper.
💡 Mnemonic: Big Charge, Laser Zaps, Toner Flies, Heat Fixes  →  Bitmap → Charge drum → Laser → Toner → Transfer → Fuse

💾 Why Do We Need Secondary Storage?

4.7.4.2

💡 RAM is volatile

Loses all data when power is off. Without secondary storage, all files disappear every time you shut down.

📁 Permanent storage

The OS, applications and user files must persist between sessions — stored on secondary storage, loaded into RAM when needed.

📦 Vastly larger capacity

Secondary storage provides terabytes of space vs a few gigabytes of RAM, storing entire libraries of programs and data.

🔋 Non-volatile

Secondary storage retains data without a power supply — unlike volatile RAM which needs constant power.

Key terms Non-volatile = keeps data without power.    Volatile = loses data without power (e.g. RAM, cache).

🗄️ Hard Disk Drive (HDD) — Principles of Operation

4.7.4.2

How it stores data

  • Spots on a magnetisable coating are magnetised in one direction = 0, other direction = 1
  • Multiple rigid platters stacked on a spindle — all coated in iron/cobalt-based material
  • Disk divided into concentric Tracks → each track divided into Sectors/Blocks

Reading data

  • Disk spins at high speed
  • Read/write head moves to correct track
  • Waits for correct sector to spin around
  • Senses magnetic field and converts to binary

Speed

Moderate — limited by mechanical movement (seek time + rotational delay)

⚠️ Critical fact The R/W head NEVER physically touches the platter — it floats nanometres above it on a cushion of air. Physical contact = "head crash" = data loss.
💡 Mnemonic: Magnetise one way = 0; other way = 1. Tracks → Sectors → Head reads as disk spins.

💿 Optical Disk — Principles of Operation

4.7.4.2

Track structure

Data on ONE continuous SPIRAL track (unlike HDD's concentric rings)

Pits & Lands

Pits = indentations pressed/burnt into track surface.
Lands = flat areas between pits

TRANSITION = 1

A change between land and pit (or pit and land) represents a binary 1

CONTINUATION = 0

No transition (staying on a land or staying in a pit) represents a binary 0

Reading

  • Laser fires and the disk spins
  • Light reflected back to a photodiode and converted to binary
⚠️ Very common mistake! It's the TRANSITION between pit/land that = 1. NOT "pit = 1" and "land = 0". Many students get this wrong — learn it precisely!
💡 Mnemonic: TRANSITION = 1 (a change happens). CONTINUATION = 0 (nothing changes).

⚡ Solid-State Drive (SSD) — Principles of Operation

4.7.4.2

Technology

NAND flash memory — no moving parts at all

Storage mechanism

Floating gate transistors trap electrical charge (electrons) between oxide layers.
Trapped charge = 1; no charge = 0. Retained without power — non-volatile.

Organisation

Data organised into Pages (4–16 KB). Many pages form one Block.

The critical rule

Cannot overwrite a page directly. An entire BLOCK must be erased first, then the page written. A dedicated Controller chip manages wear levelling and erase-write cycles.

⚠️ Fundamental hardware constraint You CANNOT erase individual pages — the whole BLOCK must be erased. This is a fundamental constraint of NAND flash.
💡 Mnemonic: Read any page. Write to empty pages. To overwrite: ERASE WHOLE BLOCK first, then write.
