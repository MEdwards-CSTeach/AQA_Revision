01

Core OOP Concepts

Class
Definition: A blueprint or template that defines the attributes (fields/properties) and behaviours (methods) that objects of that type will have. A class does not occupy memory in the same way an object does — it is a definition.
Example: A 'BankAccount' class defines that all bank accounts have a balance and can deposit/withdraw money.
Java
public class BankAccount {
  private double balance;
  private String owner;
  public BankAccount(String owner,
      double balance) {
    this.owner = owner;
    this.balance = balance;
  }
}
Python
class BankAccount:
  def __init__(self,
      owner, balance):
    self.__balance = balance
    self.__owner = owner
Object
Definition: A specific instance of a class, created at runtime. Each object has its own copy of the attributes defined by the class, but shares the methods defined in the class.
Example: myAccount = new BankAccount('Alice', 1000.0) creates a specific object from the BankAccount class.
Java
// Creating objects (instances)
BankAccount myAccount =
  new BankAccount("Alice", 1000.0);
BankAccount yourAccount =
  new BankAccount("Bob", 500.0);
Python
# Creating objects (instances)
my_account = BankAccount(
    'Alice', 1000.0)
your_account = BankAccount(
    'Bob', 500.0)
Instantiation
Definition: The process of creating a specific object (instance) from a class definition. This typically involves calling a constructor, which sets up the initial state of the object.
Example: Calling new Dog('Rex', 'Labrador') instantiates a Dog object.
Constructor
Definition: A special method called when an object is instantiated. It initialises the object's attributes. In Java, it has the same name as the class. In Python, it is __init__.
Java
public class Student {
  private String name;
  private int age;
  // Constructor
  public Student(String name,
      int age) {
    this.name = name;
    this.age = age;
  }
}
Python
class Student:
  def __init__(self,
      name, age):
    self.name = name
    self.age = age

s = Student('Alice', 16)
Attribute / Field / Property
Definition: A variable that belongs to an object or class, storing data that describes the state of the object. Also called instance variables or member variables.
Example: A Car class might have attributes: colour, speed, fuelLevel.
Method
Definition: A function defined inside a class that describes the behaviour of objects. Methods can access and modify the object's attributes. In Python, methods always take 'self' as the first parameter.
Java
public void deposit(double amount) {
  balance += amount;
}
public double getBalance() {
  return balance;
}
Python
def deposit(self, amount):
  self.__balance += amount

def get_balance(self):
  return self.__balance
02

Encapsulation & Access Control

Encapsulation
Definition: The bundling of data (attributes) and methods that operate on that data within a single unit (class) and restricting direct access to some of the object's components. This protects internal state and hides implementation details.
Example: A BankAccount hides its balance behind private access — external code cannot change the balance directly but must use deposit() or withdraw().
Access Specifiers / Modifiers
Definition: Keywords that control the visibility and accessibility of class members.
ModifierAccess ScopeJavaPython Convention
publicAccessible from anywherepublicself.name
privateAccessible only within the classprivateself.__name
protectedAccessible within class and subclassesprotectedself._name
Getter and Setter Methods
Definition: Getter methods (accessors) return the value of a private attribute. Setter methods (mutators) allow controlled modification of a private attribute, often with validation. These are how encapsulation allows safe access to private data.
Example: getBalance() returns the balance; setAge(int a) sets age only if a > 0.
Java
public int getAge() { // getter
  return age;
}
public void setAge(int a) { // setter
  if (a > 0) age = a;
}
Python
def get_age(self):    # getter
  return self.__age

def set_age(self, a): # setter
  if a > 0:
    self.__age = a
03

Inheritance

Inheritance
Definition: A mechanism where a class (subclass/child class) derives attributes and methods from another class (superclass/parent class). The subclass can extend or override the inherited behaviour. Promotes code reuse.
Example: A 'SavingsAccount' class inherits from 'BankAccount', gaining all its attributes and methods, and adds an interestRate attribute.
Java
public class Animal {
  private String name;
  public void eat() { ... }
}
// Dog inherits from Animal
public class Dog
    extends Animal {
  public void bark() {
    System.out.println("Woof!");
  }
}
Python
class Animal:
  def eat(self):
    print(f'{self.name} eats.')

# Dog inherits from Animal
class Dog(Animal):
  def bark(self):
    print('Woof!')
Superclass / Parent Class / Base Class
Definition: The class that is inherited from. It provides common attributes and methods that subclasses can inherit and potentially override.
Example: In 'class Dog extends Animal', Animal is the superclass.
Subclass / Child Class / Derived Class
Definition: A class that inherits from a superclass. It gains the superclass's attributes and methods and can add new ones or override existing ones.
Example: Dog is a subclass of Animal: it inherits eat() and breathe() but also adds bark().
The super Keyword
Definition: Used in a subclass to refer to the superclass. Commonly used to call the parent class constructor or to call an overridden method from the parent.
Java
public SavingsAccount(
    String owner, double balance,
    double rate) {
  super(owner, balance); // parent
  this.interestRate = rate;
}
Python
def __init__(self, owner,
    balance, rate):
  # call parent constructor
  super().__init__(owner, balance)
  self.interest_rate = rate
04

Polymorphism & Overriding

Polymorphism
Definition: The ability of different classes to respond to the same method call in different ways. A method call on a superclass reference can behave differently depending on the actual type of the object at runtime. "Many forms".
Example: Different Animal subclasses each override makeSound() differently — Dog barks, Cat meows, Cow moos — but all called via the same method name.
Java
Animal a = new Dog();
a.makeSound(); // prints Woof!

Animal b = new Cat();
b.makeSound(); // prints Meow!
Python
animals = [Dog(), Cat()]
for a in animals:
  a.make_sound()
  # Woof! then Meow!
Overriding
Definition: When a subclass provides its own implementation of a method that is already defined in its superclass. The subclass version replaces the superclass version for objects of the subclass type.
Java
@Override // overrides Shape
public double area() {
  return Math.PI * radius * radius;
}
Python
# overrides Shape
def area(self):
  import math
  return math.pi * self.radius**2
Abstract Method
Definition: A method declared in a class but with no implementation. Subclasses must provide their own implementation. A class with abstract methods is an abstract class and cannot be instantiated directly.
Example: An abstract Shape class declares area() as abstract — Rectangle and Circle must each provide their own implementation.
Virtual Method
Definition: A method in a superclass that can be overridden in subclasses. In Java, all non-static, non-final methods are virtual by default. The correct version is resolved at runtime (dynamic dispatch).
Static Method
Definition: A method that belongs to the class itself rather than to any specific object (instance). It can be called without creating an object. Static methods cannot access instance attributes or other non-static methods directly.
Java
public static int square(int n) {
  return n * n;
}
// No object needed:
MathHelper.square(5); // 25
Python
@staticmethod
def square(n):
  return n * n

# No object needed:
MathHelper.square(5) # 25
05

Relationships Between Classes

Aggregation ◇ white/hollow diamond
Definition: A 'has-a' relationship between classes where the contained object can exist independently of the container. If the container is destroyed, the contained object can still exist.
Example: A Library has Books. But if the Library closes, the Books still exist — Books can exist outside the Library.
Composition ◆ black/filled diamond
Definition: A stronger 'has-a' relationship where the contained object cannot exist without the container — "is made up of". If the container is destroyed, the contained objects are also destroyed.
Example: A House has Rooms. If the House is demolished, the Rooms cease to exist. A Room cannot exist without a House.
06

OOP Design Principles

Encapsulate What Varies

Identify the parts of a program that change frequently and separate them from what stays the same. Isolate varying behaviour into its own class so changes don't affect the rest of the code.

Example: Instead of putting tax calculation logic directly in an Order class, extract it into a TaxCalculator class. When tax rules change, only that class needs updating.

Favour Composition Over Inheritance

Where possible, assemble objects from smaller, reusable components (composition) rather than building large inheritance hierarchies. This leads to more flexible, maintainable code.

Example: Rather than creating Duck inheriting from Bird and FlyingBird, give Duck a FlyBehaviour object that can be swapped at runtime.

Program to Interfaces, Not Implementation

Write code that depends on abstract types (interfaces or abstract classes) rather than concrete implementations. This allows the implementation to be swapped without changing the code that uses it.

Example: A program that uses a Shape interface with area() can work with any shape (Circle, Rectangle) without knowing the specific type.
