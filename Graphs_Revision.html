Section 1

What is a Graph?

A graph is a fundamental data structure for modelling complex relationships between objects.

AQA Definition: A graph is a data structure consisting of a set of vertices (nodes) connected by a set of edges (arcs). Graphs are used to represent more complex relationships than trees can model.
Vertex / Node
A point in the graph representing an object, place, state or entity
Edge / Arc
A connection (line) between two vertices representing a relationship
Degree
The number of edges connected to a vertex. In a directed graph: in-degree (arriving) and out-degree (leaving)
Path
A sequence of vertices connected by edges, with no vertex repeated
Cycle
A path that starts and ends at the same vertex — graphs CAN have cycles; trees cannot
Connected
A graph is connected if every vertex can be reached from every other vertex

Section 2

Types of Graph

Graphs come in several varieties — knowing which type applies in a given scenario is essential.

Undirected Graph: Edges have no direction. If vertex A is connected to B, then B is also connected to A. The relationship is symmetric.
  • Edges drawn as plain lines (no arrows)
  • Adjacency matrix is always symmetric — matrix[i][j] = matrix[j][i]
  • Each edge adds 1 to both endpoints' degree
  • Examples: Facebook friendships, peer-to-peer networks, undirected road maps

A─B, A─D, B─C, B─E, C─D, D─E (undirected)

Directed Graph (Digraph): Edges have a direction shown by an arrow. A→B does NOT mean B→A.
  • Edges drawn as arrows
  • In-degree: edges arriving at a vertex
  • Out-degree: edges leaving a vertex
  • Adjacency matrix is NOT necessarily symmetric
  • Examples: Twitter follows, web hyperlinks, one-way streets, task dependencies
  • Can still contain directed cycles (A→B→C→A)

A→B, A→C, B→D, C→B, D→A (directed)

Weighted Graph: Each edge has an associated weight — a number representing cost, distance, time or capacity.
  • Can be directed OR undirected
  • Adjacency matrix stores the weight value instead of 0/1
  • No edge represented as ∞ (infinity) or 0
  • Used in shortest-path algorithms: Dijkstra's, Bellman-Ford, A*
  • Unweighted graphs treat all edges as weight = 1
  • Examples: road distances, airline costs, network bandwidth

Representing Graphs

Two standard ways to store a graph in memory — each with distinct trade-offs.

A 2D array of size V × V. Entry [i][j] = 1 (or weight) if there is an edge from vertex i to vertex j; 0 (or ∞) otherwise.

Graph: A─B, A─C, B─D, C─D

ABCD
A0110
B1001
C1001
D0110

Green = edge exists. Matrix is symmetric (undirected graph).

Properties

  • Space: O(V²) — always allocates V² entries regardless of actual edges
  • Edge lookup: O(1) — direct array index
  • Find all neighbours: O(V) — must scan entire row
  • Best for: Dense graphs where most edges exist
  • Symmetry: Undirected → matrix[i][j] = matrix[j][i]
  • Weighted: Store weight instead of 1; ∞ for no edge
Each vertex stores a list of its directly connected neighbours. Only existing edges consume memory.

Graph: A─B, A─C, B─D, C─D

A → B C
B → A D
C → A D
D → B C

Only 8 neighbour entries for 4 edges (undirected).

Properties

  • Space: O(V + E) — efficient for sparse graphs
  • Edge lookup: O(degree of vertex) — must scan list
  • Find all neighbours: O(degree) — just iterate list
  • Best for: Sparse graphs (few edges per vertex)
  • Weighted: Store (neighbour, weight) pairs
  • BFS/DFS: More efficient — only visits real edges

Adjacency Matrix – Python

class GraphMatrix:
    def __init__(self, vertices):
        self.v = vertices
        self.m = [[0]*vertices
                   for _ in range(vertices)]

    def add_edge(self, u, w, weight=1):
        self.m[u][w] = weight
        self.m[w][u] = weight  # undirected

    def has_edge(self, u, w):
        return self.m[u][w] != 0

    def neighbours(self, u):
        return [i for i in range(self.v)
                if self.m[u][i] != 0]

g = GraphMatrix(4)
g.add_edge(0, 1)  # A-B
g.add_edge(0, 2)  # A-C
g.add_edge(1, 3)  # B-D
g.add_edge(2, 3)  # C-D

Adjacency List – Python

class GraphList:
    def __init__(self):
        self.adj = {}

    def add_vertex(self, v):
        if v not in self.adj:
            self.adj[v] = []

    def add_edge(self, u, v, wt=1):
        self.adj[u].append((v, wt))
        self.adj[v].append((u, wt))

    def neighbours(self, v):
        return [n for n, w in self.adj[v]]

g = GraphList()
for v in ['A','B','C','D']:
    g.add_vertex(v)
g.add_edge('A', 'B')
g.add_edge('A', 'C')
g.add_edge('B', 'D')
g.add_edge('C', 'D')

Graph Traversals

Two fundamental algorithms for visiting every vertex in a graph. Use the interactive demo below to see them step by step.

BFS — Breadth-First Search
Visits all neighbours of a vertex before moving deeper. Explores the graph level by level.
  • Uses a QUEUE (FIFO) to track vertices
  • Marks vertices visited when enqueued
  • Guarantees shortest path in an unweighted graph
  • Time: O(V + E)  |  Space: O(V)
  • AQA use: shortest path, social network distances

BFS Pseudocode

BFS(graph, start):
  queue   ← [start]
  visited ← {start}
  WHILE queue is not empty:
    v ← dequeue from front
    OUTPUT v
    FOR EACH neighbour n of v:
      IF n not in visited:
        visited.add(n)
        enqueue(n)

Uses QUEUE → level by level
→ shortest path guaranteed
DFS — Depth-First Search
Explores as far as possible along each branch before backtracking. Goes deep before going wide.
  • Uses a STACK (LIFO) — or recursion (call stack)
  • Does NOT guarantee shortest path
  • Memory: O(V) — stack holds one path at a time
  • Time: O(V + E)
  • AQA use: maze solving, cycle detection, topological sort

DFS Pseudocode

DFS(graph, start):
  stack   ← [start]
  visited ← {}
  WHILE stack is not empty:
    v ← pop from top
    IF v not in visited:
      visited.add(v)
      OUTPUT v
      FOR EACH neighbour n of v:
        IF n not in visited:
          push(n)

Uses STACK → deep first
→ maze solving, backtracking

BFS – Python

from collections import deque

def bfs(graph, start):
    visited = set()
    queue   = deque([start])
    visited.add(start)
    order   = []
    while queue:
        v = queue.popleft()  # FIFO
        order.append(v)
        for n in graph[v]:
            if n not in visited:
                visited.add(n)
                queue.append(n)
    return order

# graph = {'A':['B','C'],
#           'B':['A','D','E'],
#           'C':['A','F'], ...}
# bfs(graph, 'A') → ['A','B','C',...]

DFS – Python (iterative & recursive)

def dfs(graph, start):
    visited = set()
    stack   = [start]
    order   = []
    while stack:
        v = stack.pop()  # LIFO
        if v not in visited:
            visited.add(v)
            order.append(v)
            for n in reversed(graph[v]):
                if n not in visited:
                    stack.append(n)
    return order

# Recursive version:
def dfs_rec(graph, v, vis=None):
    if vis is None: vis = set()
    vis.add(v); print(v)
    for n in graph[v]:
        if n not in vis:
            dfs_rec(graph, n, vis)

Graph vs Tree

A tree IS a graph — but with strict additional constraints. This distinction is frequently examined.

AQA Specification definition: "A tree is a connected, undirected graph with no cycles." — All trees are graphs, but not all graphs are trees.

✓ This IS a tree — connected, no cycles, N−1 edges

✗ This is NOT a tree — it contains a cycle

Three constraints that make a graph a tree

1. Connected

Every vertex must be reachable from every other vertex. A disconnected graph — even with no cycles — is NOT a tree.

2. No Cycles (Acyclic)

There must be absolutely no cycles. Adding even one extra edge to a tree immediately creates a cycle → no longer a tree.

3. N−1 Edges

For N vertices, a tree has exactly N−1 edges. This follows automatically from being connected and acyclic.

Section 6

Real-World Applications

Graphs model almost any networked system. Here are the most exam-relevant applications.

GPS Navigation

Road network as a weighted directed graph. Dijkstra's or A* finds the shortest/fastest route. Edge weights = distance or time.

Social Networks

Users = vertices, friendships/follows = edges. BFS finds "degrees of separation". Graph clustering finds communities.

Internet Routing

Routers = vertices, cables = weighted edges. OSPF and BGP routing protocols use graph algorithms to find optimal paths.

Maze Solving

Cells = vertices, open passages = edges. DFS naturally solves mazes by exploring one path fully before backtracking.

Compiler / Build Tools

Task dependency graphs detect circular imports. Topological sort orders compilation steps. Used in Make, Gradle, npm.

Game AI

Game map as a graph. BFS finds shortest escape route. A* (heuristic BFS) used in pathfinding for NPCs in real games.
